# Tmux AI 工作流实战

> B站视频总结，介绍如何用 Tmux 同时运行多个 AI 编码任务

## 核心价值

**Tmux 让你能同时开多个 AI 写代码**，管理多进程进度。

## 三种关键 API

### 1. 终端操控 API

- AI 可以操控 Tmux 里的终端
- 查看终端内容、发送按键
- **让 AI 同时运行多个进程**

### 2. 窗口布局 API

- 一键创建分屏布局
- 工作区自动化（创建 Git 分支 → 创建对应窗口）
- 左边放 AI，右上 Git，右下项目运行

### 3. Hooks（事件监听）

- 监听分屏切换、窗口关闭、工作区重命名等事件
- **自动切换浏览器到对应项目的运行效果**
- AI 完成工作时发送通知

## 实战场景

### 多 AI 并行开发

```
脚本一键创建工作区
├── 新 Git 分支
├── 新 Tmux 窗口（命名=分支名）
├── 左：AI 终端
├── 右上：Git
└── 右下：项目运行
```

### 进度监控面板

- 快捷键呼出，显示所有 AI 进度
- 完成的工作区自动高亮
- 一键跳转到对应工作区

## 配置要点

- **Prefix 键**：Ctrl+S（避免与其他软件冲突）
- **键盘布局**：JKLI 作为上下左右（Comet 键盘）
- **会话管理**：Ot+数字切换，Ot+Q 关闭
