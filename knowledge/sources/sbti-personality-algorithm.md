# SBTI 人格测试算法分析

> 来源：量子位 | 日期：2026-04-11
> 对虚拟用户 Persona 构建有参考价值

## 核心算法逻辑

### 三层架构
```
答题拆维度 → 分数归档 → 模板匹配
```

### 1. 维度设计
- 5个模型：自我(S)、情感(E)、态度(A)、驱力(Ac)、社交(So)
- 每个模型3个子维度 → **共15个维度**
- 每维度2题，每题1-3分 → 维度总分2-6分

### 2. 分数归档
| 分值 | 档位 |
|------|------|
| 2-3 | L (低) |
| 4 | M (中) |
| 5-6 | H (高) |

### 3. 模板匹配
- 用户向量：15维 (如 H,M,H,L,M...)
- 预设模板：25种人格类型
- 匹配方式：向量距离 + 精确命中数

## 对虚拟用户研究的启发

| 借鉴点 | 应用思路 |
|--------|----------|
| 多维度人格建模 | 虚拟用户用15维向量表示 |
| 分数归档 | 连续值→离散档位，减少匹配复杂度 |
| 向量距离匹配 | 找最接近的真实用户群体作为模板 |
| 特殊判定机制 | 触发条件进入特殊分支 |
