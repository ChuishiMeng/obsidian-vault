# Quick BI 正式开放 CLI 工具，让数据操作迈入 AI 原生时代

> 来源：瓴羊 Quick BI 和 Dataphin 公众号
> 日期：2026-04-07

---

## 核心变化

Quick BI 把开放平台的 **134 项 API 功能**全部转化为终端命令，开发者（或 AI Agent）可以用一行自然语言完成原来需要写大量代码的操作。

---

## 六大能力模块

组织管理、权限管理、资源管理、数据管理、嵌入分析、统计分析 — 能力不裁剪，使用更轻量。

---

## 四个高频场景

| 场景 | 原来 | 现在 |
|------|------|------|
| **报表外嵌** | 写代码生成 Ticket + 定时续期，跨天开发 | 一句话开通嵌入，秒级完成 |
| **数据资产盘点** | 手动翻页面或写复杂 API 调用 | 一句话获取 Top10 报表 + 性能分析 |
| **用户/权限批量管理** | 后台逐一操作 | 一句话完成入职 + 权限分配 |
| **问数 Skill 配合** | 批量赋权流程繁琐 | CLI 赋权 → Skill 问数，一条龙 |

---

## AI Agent 集成

文中明确提到了 **OpenClaw、Qoder work、Claude Code、悟空** 等主流 Agent，提供了专门的 AI 安装指南：

```
https://quickbi-prod.oss-cn-beijing.aliyuncs.com/quickbi-cli/AI_INSTALLATION_GUIDE.md
```

安装后可以直接给 Agent 发自然语言：
- "给我展示近30天访问量前10的报表"
- "把销售大盘开通嵌入访问权限，发给xx总"

---

## 意义

这是 BI 工具从"人操作界面"向"AI Agent 操作 CLI"转型的典型案例。对于数据问答产品来说，这种 Agent + BI CLI 的模式值得参考。
