## 轨迹数据定义

**轨迹 (Trajectory)**：时间戳标注的空间点序列
```
Trajectory = {p1, p2, ..., pn}
其中 pi = (latitude, longitude, timestamp, [attributes])
```

## GeoLife 数据集

| 属性 | 值 |
|------|-----|
| 用户数 | 182 |
| 时间跨度 | 2007.4 - 2012.8 (3年+) |
| 轨迹数 | 17,621 |
| 总距离 | 120万公里 |
| 总时长 | 48,000+ 小时 |
| 采样率 | 91%为1-5秒或5-10米 |

**活动类型**：日常生活（通勤）、娱乐、体育（徒步、骑行）

## 轨迹数据挖掘任务分类

### 1. 轨迹预处理 (Trajectory Preprocessing)
- 噪声过滤
- 轨迹分段
- 地图匹配
- 压缩

### 2. 轨迹模式挖掘 (Trajectory Pattern Mining)
- **频繁模式**：发现常见路径
- **周期模式**：发现周期性行为
- **聚类**：相似轨迹分组
- **序列模式**：发现访问顺序

### 3. 轨迹分类 (Trajectory Classification)
- 交通方式识别（步行、驾车、公交）
- 用户身份识别
- 活动类型识别

### 4. 轨迹异常检测 (Trajectory Anomaly Detection)
- 偏离正常路线
- 异常停留点
- 异常速度

### 5. 轨迹预测 (Trajectory Prediction)
- 下一个位置预测
- 目的地预测
- 路径预测

## 关键技术

### 地图匹配 (Map Matching)
```
GPS点 → 道路段
挑战：GPS误差、道路网络复杂性
方法：隐马尔可夫模型 (HMM)、几何匹配
```

### 语义增强 (Semantic Enrichment)
```
原始轨迹 → 语义轨迹
例如：(lat, lon, t) → (家→公司，08:00，驾车)
需要：POI数据库 + 时间上下文
```

### 时空索引 (Spatio-Temporal Indexing)
- R-tree 变体
- Grid-based
- 时空四叉树

## 与虚拟用户结合点

### 1. 生成虚拟轨迹
**需求**：为虚拟用户生成真实感的城市移动轨迹

**方法**：
- 基于真实数据学习模式
- 个性化：考虑虚拟用户画像
- 约束：道路网络、时间规律

**应用**：
- 城市模拟实验
- 交通算法测试
- 应急演练

### 2. 轨迹与身份关联
**郑宇的发现**：轨迹可以识别用户身份和行为

**虚拟用户应用**：
- 确保虚拟用户轨迹与其人口统计特征一致
- 验证：轨迹模式是否符合虚拟角色设定

### 3. 语义行为理解
**从轨迹到活动**：
```
轨迹 + 时间 + POI → 活动类型
例：工作日早上9点到达写字楼 → 上班
```

**虚拟用户需求**：
- 虚拟用户的活动应该有语义解释
- 轨迹生成应该基于活动语义

## 学习收获

1. **轨迹是城市计算的"血液"**：反映人流、车流、资源流
2. **从坐标到语义**：需要融合POI、时间、上下文
3. **模式与异常**：正常行为建模是异常检测的基础
4. **预测价值**：轨迹预测支持智能服务
