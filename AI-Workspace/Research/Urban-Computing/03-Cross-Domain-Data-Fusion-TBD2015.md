# 论文学习：跨域数据融合方法论

> 基础论文：Methodologies for Cross-Domain Data Fusion: An Overview
> 作者：Yu Zheng
> 发表：IEEE Transactions on Big Data, 2015
> 
> 深度学习扩展：Deep Learning for Cross-Domain Data Fusion in Urban Computing
> 作者：Zou et al. (含Yu Zheng)
> 发表：Information Fusion, 2025

## 核心问题

**跨域数据融合**：如何将来自不同源、不同模态、不同表示的数据整合，以解决单一数据源无法处理的复杂城市问题。

> "Traditional data mining usually deals with data from a single domain. In the era of big data, the volume of data from different domains increases rapidly."

---

## 经典方法论（2015）

### 三大融合范式

```
┌──────────────────────────────────────────────────────────────┐
│                Cross-Domain Data Fusion 范式                  │
├──────────────────────────────────────────────────────────────┤
│  1. Stage-based（阶段式融合）                                 │
│     └─ 多阶段处理：先用数据A，再用数据B                         │
│                                                                │
│  2. Feature-level-based（特征级融合）                          │
│     └─ 将不同数据的特征拼接/融合                               │
│                                                                │
│  3. Semantic-based（语义级融合）                               │
│     ├─ Multi-view Learning（多视图学习）                       │
│     ├─ Similarity-based（基于相似性）                          │
│     ├─ Probabilistic Dependency（概率依赖）                    │
│     └─ Transfer Learning（迁移学习）                           │
└──────────────────────────────────────────────────────────────┘
```

### 应用案例

| 应用 | 数据源1 | 数据源2 | 融合方式 |
|------|---------|---------|---------|
| 空气质量推断 | 监测站数据 | 气象+POI+交通 | 多视图学习 |
| 城市功能区发现 | POI | 人流轨迹 | 主题模型 |
| 交通时间预测 | GPS轨迹 | 道路网络 | 特征融合 |

## 深度学习时代方法论（2024扩展）

### 四大融合范式

```
┌──────────────────────────────────────────────────────────────┐
│            Deep Learning Data Fusion 范式（2024）             │
├──────────────────────────────────────────────────────────────┤
│  1. Feature-based Fusion（特征融合）                          │
│     ├─ Early Fusion: 输入层融合                               │
│     ├─ Late Fusion: 决策层融合                                │
│     └─ Hybrid Fusion: 混合融合                                │
│                                                                │
│  2. Alignment-based Fusion（对齐融合）                         │
│     ├─ Temporal Alignment: 时间对齐                           │
│     ├─ Spatial Alignment: 空间对齐                            │
│     └─ Cross-modal Alignment: 跨模态对齐                      │
│                                                                │
│  3. Contrast-based Fusion（对比融合）                          │
│     └─ 对比学习：拉近相似样本，推远不相似样本                   │
│                                                                │
│  4. Generation-based Fusion（生成融合）                        │
│     ├─ GAN-based: 生成对抗网络                                │
│     ├─ Diffusion-based: 扩散模型                              │
│     └─ LLM-based: 大语言模型                                  │
└──────────────────────────────────────────────────────────────┘
```

## 数据类型与模态

### 按数据源分类

| 类型 | 描述 | 示例 |
|------|------|------|
| 地理数据 | 位置、POI、路网 | 经纬度、兴趣点 |
| 交通数据 | 流量、速度、轨迹 | 出租车GPS、公交数据 |
| 社交媒体 | 用户生成内容 | Twitter、微博签到 |
| 人口统计 | 人口特征 | 年龄、收入、教育 |
| 环境数据 | 自然环境信息 | 气象、空气质量 |
| 其他数据 | 其他城市数据 | 医疗、能耗数据 |

### 按模态分类

| 模态 | 数据形式 | 处理方法 |
|------|----------|----------|
| 时空数据 | 时间序列+空间位置 | RNN, GNN, Transformer |
| 视觉数据 | 图像、视频 | CNN, ViT |
| 文本数据 | 社交媒体、报告 | BERT, LLM |
| 图数据 | 网络、关系 | GNN |

## 与虚拟用户研究的结合点

### 1. 多模态虚拟用户生成

**场景**：生成具备丰富属性的虚拟用户

| 数据模态 | 虚拟用户属性 |
|----------|-------------|
| 时空数据 | 移动轨迹、活动范围 |
| 人口统计 | 年龄、职业、收入 |
| 社交数据 | 兴趣、社交网络 |
| 行为数据 | 消费、出行习惯 |

**融合方法**：使用特征级融合或生成式融合

### 2. 虚拟用户与城市环境交互

**场景**：虚拟用户在城市中的行为模拟

```
虚拟用户 + 城市环境数据 → 行为预测
     ↓
[人口统计] + [POI分布] + [交通网络]
     ↓
预测：出行方式、活动地点、消费行为
```

### 3. 跨域知识迁移

**场景**：将真实用户的行为模式迁移到虚拟用户

- **源域**：真实用户轨迹数据
- **目标域**：虚拟用户行为生成
- **方法**：迁移学习、领域适应

### 4. LLM增强的虚拟用户

**潜力**：
- 用LLM生成虚拟用户的语义描述
- 多模态融合：文本描述 + 结构化属性
- 城市知识注入：将城市知识图谱融入LLM

## 关键洞察

### 方法论演进

| 时代 | 主导方法 | 特点 |
|------|----------|------|
| 2015前 | 传统机器学习 | 手工特征、规则融合 |
| 2015-2020 | 深度学习初期 | CNN/RNN自动特征学习 |
| 2020-2024 | 深度学习成熟 | 对比学习、预训练模型 |
| 2024+ | LLM时代 | 大模型、生成式融合 |

### 核心挑战

1. **数据异构性**：不同模态数据如何对齐
2. **时空依赖**：如何建模复杂的时空关系
3. **数据稀疏**：部分区域/模态数据不足
4. **可解释性**：融合决策如何解释

---

## 学习收获

1. **跨域融合是城市计算的核心**：单一数据源难以解决复杂城市问题
2. **方法论不断演进**：从阶段式到深度学习再到LLM
3. **虚拟用户天然需要多模态**：属性、行为、环境需要融合
4. **LLM带来新机遇**：生成式方法可能改变虚拟用户构建方式
