## 🏗️ 三层图谱结构 (HKSG)

### L0: 元模型层

**作用**：定义结构规范，约束 L1 和 L2。

```
元模型定义:
- EntityType: 实体类型
- RelationType: 关系类型
- Attribute: 属性
- Constraint: 约束
```

### L1: Ontology 层

**作用**：表示业务领域的实体、关系和术语。

```
业务实体示例:
- Member (会员): 金牌会员 = level = 'gold'
- Product (商品): 热销商品 = sales_count > 1000
- Order (订单): 大额订单 = total_amount > 10000
```

### L2: Schema 层

**作用**：表示数据库的表、字段和外键关系。

```
表映射:
- Member → members
- Product → products
- Order → orders
```

## 🔍 联合召回算法

### 算法流程

```
Step 1: 问题理解
  - 实体识别
  - 意图识别
  - 术语映射

Step 2: L1 召回 (Ontology)
  - 实体召回
  - 关系召回
  - 术语展开

Step 3: L2 召回 (Schema)
  - 表映射
  - 字段映射
  - 外键补全

Step 4: 元路径约束验证
  - 构建元路径
  - 语义验证
  - 路径剪枝

Step 5: 最小完备集提取
  - 完备性检查
  - 最小化剪枝
```

---

## 🧭 元路径约束推理

### 元路径定义

```
元路径 P = (E_1, R_1, E_2, R_2, ..., E_n)

示例:
Member → Purchase → Product
  = 会员购买商品
```

### 约束规则

1. **语义一致性**: 路径必须覆盖问题中所有实体
2. **最短路径优先**: 多条路径选择最短
3. **完备性保证**: 必须满足 SQL 生成需求

---

## 📦 最小完备集提取

### 目标

在保证 SQL 可生成的前提下，最小化召回的 Schema 项。

### 示例

```
问题: "查询所有金牌会员的消费总额"

最小完备集:
- 表: members, orders
- 字段: id, level, member_id, total_amount
```
