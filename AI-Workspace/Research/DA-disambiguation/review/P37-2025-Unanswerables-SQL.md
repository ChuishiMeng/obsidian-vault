## 一、研究背景

### 1.1 问题场景

**场景描述**：用户问"为什么 TP53 基因会导致癌症？"，这不是 SQL 能回答的问题，但 Text-to-SQL 系统仍会生成 SQL，返回错误结果。

**具体示例**：
```
不可回答问题："EGFR 基因的 3D 蛋白结构是什么？"

错误行为：生成 SQL 查询不存在的列
正确行为：识别为"不可回答"，提示用户

风险场景：
- 医疗领域：错误诊断信息
- 金融领域：误导性分析
- 企业决策：基于错误数据
```

### 1.2 学术场景 vs 企业场景对比

| 维度 | 学术基准 | 企业实际 |
|------|---------|---------|
| 问题类型 | 全部可回答 | 混合（可回答+不可回答） |
| 系统行为 | 生成 SQL | 需要判断边界 |
| 错误影响 | 标注错误 | 误导决策 |

## 三、方法论

### 3.1 整体框架

```
用户问题 → NAR 规则检测 → 分类
                ↓
         8 类不可回答类型：
         1. Non-SQL
         2. Columns Missing
         3. Values Missing
         4. Out of Domain
         5. Column Ambiguous
         6. Value Ambiguous
         7. Contextual Ambiguous
         8. Operator Ambiguous
                ↓
         返回不可回答 + 解释
```

### 3.2 核心方法

#### 方法1：8 类不可回答分类

| 类型 | 描述 | 示例 |
|------|------|------|
| **Non-SQL** | 无法用 SQL 回答 | "为什么基因会导致癌症？" |
| **Columns Missing** | 缺少必要的列 | "EGFR 的 3D 蛋白结构？" |
| **Values Missing** | 数据库中不存在的值 | "列出火星上的基因" |
| **Out of Domain** | 需要外部知识 | "诺奖研究提到的基因？" |
| **Column Ambiguous** | 多列可能匹配 | "EGFR 评分是多少？" |
| **Value Ambiguous** | 值含义不明确 | "与生长相关的基因" |
| **Contextual Ambiguous** | 缺少上下文 | "导致它的基因？" |
| **Operator Ambiguous** | 操作符不明确 | "哪些更可靠？" |

#### 方法2：NAR 规则 + Few-shot Prompt

**输入**：用户问题 + Schema
**输出**：可回答/不可回答 + 类型
**核心步骤**：
1. 应用 NAR 规则检测
2. Few-shot Prompt 分类
3. 返回结果 + 解释

**关键代码**：
```python
NAR_RULES = {
    "non_sql": "问题需要解释性回答，非结构化查询",
    "column_missing": "问题引用的列不存在于 schema",
    "value_missing": "问题依赖的数据值不存在",
    "out_of_domain": "问题需要外部领域知识",
    ...
}
```

### 3.3 技术细节

| 组件 | 实现方式 | 备注 |
|------|---------|------|
| NAR 规则 | 领域专家定义 | 可扩展 |
| Few-shot | 5 可回答 + 5 不可回答 | 平衡样本 |
| 数据集 | OncoMX-NAQ | 80 个问题 |

## 四、实验结果

### 4.1 实验设置

**数据集**：

| 数据集 | 规模 | 用途 |
|--------|------|------|
| OncoMX-NAQ | 80 个不可回答问题 | 主实验 |

### 4.2 主实验结果

| 类别 | 检测准确率 | 难度 |
|------|----------|------|
| Non-SQL | ~1.0 | 低 |
| Columns Missing | ~1.0 | 低 |
| Out of Domain | ~1.0 | 低 |
| Contextual Ambiguous | ~0.8 | 中 |
| Operator Ambiguous | ~0.7 | 中 |
| Value Missing | ~0.5 | 高 |
| Value Ambiguous | ~0.5 | 高 |
| Column Ambiguous | ~0.3 | 高 |

### 4.3 关键发现

1. **结构化类型易检测**：Non-SQL、Columns Missing、Out of Domain ~100%
2. **值相关类型难检测**：Value Missing、Value Ambiguous ~50%
3. **列模糊最困难**：需要深度理解 schema
