**标题**: Disambiguate First, Parse Later: Generating Interpretations for Ambiguity Resolution in Semantic Parsing  
**作者**: Irina Saparina, Mirella Lapata (University of Edinburgh)  
**发表**: ACL 2025 Findings  
**代码**: github.com/saparina/disambiguate-then-parse

---

## 1. 两阶段消歧方法

### 核心流程

```
阶段 I: 消歧 (Disambiguation)
  步骤1: 默认解释生成 (LLM Prompt)
    输入：问题 + 数据库 Schema
    输出：初始解释集 {û₁, û₂, ...}
  
  步骤2: 解释填充 (微调模型)
    输入：问题 + 初始解释
    输出：补充解释集
    关键：通过 SQL 执行结果验证解释完整性
                    ↓
阶段 II: Text-to-SQL 解析
  将每个无歧义解释映射到 SQL 查询
```

### 方法优势

| 优势 | 说明 |
|------|------|
| **模块化设计** | 消歧和解析可独立优化 |
| **利用现有模型** | 解析阶段可使用标准 text-to-SQL 模型 |
| **透明性** | 自然语言解释使用户理解推理过程 |
| **处理模糊性** | 明确处理多种歧义类型 |

## 2. 自然语言解释生成方法

### 默认解释生成

- 使用 **Llama-3.1 8B** 零样本提示
- 利用 LLM 固有 bias 生成"默认"解释
- 无需额外训练

### 解释填充模型

**关键创新**: 不直接比较自然语言，而是**比较 SQL 执行结果**

训练流程：
1. 生成默认解释集 → 解析为 SQL
2. 执行 SQL，与黄金 SQL 结果比较
3. 识别缺失解释（执行结果不同）
4. 训练填充模型补充缺失解释

## 3. 实验结果

### AmbiQT 和 Ambrosia 数据集

| 方法 | AmbiQT Single | AmbiQT Full | Ambrosia Single | Ambrosia Full |
|------|---------------|-------------|-----------------|---------------|
| 端到端 0-shot | 62.3% | 12.3% | 29.4% | 0.9% |
| 端到端 SFT | 82.1% | 63.2% | 38.0% | 0.4% |
| **Ours** | **92.3%** | **53.2%** | **84.4%** | **18.8%** |

### 关键发现

1. **消歧优先策略有效**: 域内外都优于端到端方法
2. **Infilling 提升显著**: Full Coverage 从 26% 提升到 53%
3. **泛化能力强**: 不同标注风格、数据库结构下保持一致性能
4. **全解释覆盖仍有挑战**: Ambrosia 上仅 19%

---

## 4. 与其他方法对比

| 维度 | 端到端生成 SQL | 本文方法 |
|------|---------------|----------|
| **流程** | 问题 → 多个 SQL | 问题 → 解释 → SQL |
| **可解释性** | 低 (黑盒) | 高 (自然语言解释) |
| **模块化** | 低 | 高 |
| **域外泛化** | 低 (0.4-0.9%) | 高 (16.9-18.8%) |

---

## 5. 局限性

1. Full Coverage 在 Ambrosia 上仅 19%
2. 75% 错误来自默认解释生成阶段
3. 计算成本高于端到端方法
