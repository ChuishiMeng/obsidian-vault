```python
def generate_user_variations(base_query, user_profile):
    """
    基于用户画像生成表达变体
    
    参数:
        base_query: 标准查询
        user_profile: 用户画像模板
    
    返回:
        List[variation]
    """
    variations = []
    
    # 变体 1: 口语化
    if user_profile['style'] == 'colloquial':
        variations.append(colloquialize(base_query))
        # "查销售额" → "帮我看看卖得咋样"
    
    # 变体 2: 简称/黑话
    if user_profile['style'] == 'abbreviated':
        variations.append(abbreviate(base_query))
        # "数字科技控股" → "数科"
    
    # 变体 3: 指代省略
    if user_profile['style'] == 'context_dependent':
        variations.append(omit_context(base_query))
        # "查它的销售额" → "它" 指代不明
    
    # 变体 4: 角色视角
    if user_profile['role'] == 'sales_director':
        variations.append(add_role_perspective(base_query, 'aggregate'))
        # "查销售额" → 期望汇总数据
    
    return variations
```

### 2.2 准确性指标

#### 2.2.1 消歧准确率 (Disambiguation Accuracy)

**定义**：消歧决策正确的查询比例。

$$
\text{DA-Accuracy} = \frac{|\{q \in Q \mid \text{disambig}(q) = \text{gold}(q)\}|}{|Q|}
$$

**分层准确率**：
$$
\text{DA-Accuracy}_{\text{layer}} = \frac{|\{q \in Q_{\text{layer}} \mid \text{disambig}(q) = \text{gold}(q)\}|}{|Q_{\text{layer}}|}
$$

其中 layer ∈ {user, business, data, multi-layer}

**分级准确率**：
| 级别 | 定义 | 计算方式 |
|------|------|----------|
| **Exact Match** | 所有歧义均正确 | 所有歧义类型完全匹配 |
| **Partial Match** | ≥50% 歧义正确 | 正确歧义数/总歧义数 ≥ 0.5 |
| **Any Correct** | 至少一个歧义正确 | 正确歧义数 ≥ 1 |

#### 2.2.2 SQL 执行准确率 (Execution Accuracy)

**定义**：生成的 SQL 执行结果与标准答案一致的比例。

$$
\text{EA} = \frac{|\{q \in Q \mid \text{execute}(\text{SQL}_{\text{gen}}) = \text{execute}(\text{SQL}_{\text{gold}})\}|}{|Q|}
$$

**说明**：
- 比较执行结果集（而非 SQL 文本）
- 允许 SQL 语法差异，只要结果相同
- 处理浮点数精度问题 (允许 1% 误差)

#### 2.2.3 歧义召回率 (Ambiguity Recall)

**定义**：系统正确识别出的歧义占真实歧义的比例。

$$
\text{Ambiguity-Recall} = \frac{|\text{Detected Ambiguities} \cap \text{Gold Ambiguities}|}{|\text{Gold Ambiguities}|}
$$

**歧义检出 F1**：
$$
\text{Ambiguity-F1} = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}
$$

### 2.3 效率指标

#### 2.3.1 平均交互轮次 (Average Interaction Rounds)

**定义**：完成一次消歧所需的平均确认轮次。

$$
\text{AIR} = \frac{\sum_{q \in Q_{\text{ambiguous}}} \text{rounds}(q)}{|Q_{\text{ambiguous}}|}
$$

**分场景统计**：
| 场景 | 定义 | 目标值 |
|------|------|--------|
| 无歧义 | 直接执行 | 0 轮 |
| 单层歧义 | 单一层面歧义 | ≤1 轮 |
| 多层歧义 | 多层歧义叠加 | ≤3 轮 |
| 困难样本 | 高相似度候选 | ≤5 轮 |

#### 2.3.2 响应时间 (Response Time)

**定义**：从用户查询到系统响应的延迟。

| 阶段 | 目标延迟 | 说明 |
|------|----------|------|
| 歧义检测 | < 100ms | LLM 分类 + 规则匹配 |
| 三层推理 | < 500ms | 并行推理 |
| 证据融合 | < 50ms | 加权计算 |
| 问题生成 | < 300ms | EIG + LLM 重写 |
| **总计** | < 1s | 端到端延迟 |

#### 2.3.3 确认问题数 (Number of Clarification Questions)

**定义**：系统生成的澄清问题总数。

**统计维度**：
- 每查询平均问题数
- 问题类型分布 (概念/时间/粒度/规则)
- 问题质量评分 (人工评估)

### 2.4 体验指标

#### 2.4.1 用户满意度 (User Satisfaction)

**测量方式**：5 点李克特量表问卷。

| 维度 | 问题示例 | 评分 |
|------|----------|------|
| **准确性** | "系统理解了我的意图" | 1-5 |
| **效率** | "交互过程很高效" | 1-5 |
| **自然度** | "问题表述很自然" | 1-5 |
| **信任度** | "我信任系统的理解" | 1-5 |
| **总体** | "整体体验满意" | 1-5 |

**计算**：
$$
\text{Satisfaction} = \frac{\sum_{i=1}^{5} \text{score}_i}{5 \times 5} \times 100\%
$$

#### 2.4.2 认知负荷 (Cognitive Load)

**测量方式**：NASA-TLX 简化版。

| 维度 | 说明 | 评分 |
|------|------|------|
| **心理需求** | 需要多少思考 | 1-10 |
| **时间压力** | 是否感到匆忙 | 1-10 |
| **努力程度** | 需要多少努力 | 1-10 |
| **挫折感** | 是否感到沮丧 | 1-10 |

**计算**：
$$
\text{Cognitive Load} = \frac{\sum_{i=1}^{4} \text{score}_i}{40} \times 100\%
$$

**目标**：< 40% (低认知负荷)

#### 2.4.3 系统信任度 (System Trust)

**测量方式**：信任度量表 (7 项)。

| 维度 | 问题 |
|------|------|
| 能力 | "系统有能力理解我的查询" |
| 可靠 | "系统理解是可靠的" |
| 透明 | "我能理解系统的决策过程" |
| 可控 | "我可以纠正系统的错误" |
| 一致 | "系统行为是一致的" |
| 诚实 | "系统诚实地表达不确定性" |
| 善意 | "系统考虑我的利益" |

### 2.5 综合指标

#### 2.5.1 DA-Score (Disambiguation Accuracy Score)

**定义**：综合准确性和效率的单一指标。

$$
\text{DA-Score} = \alpha \times \text{DA-Accuracy} + (1 - \alpha) \times (1 - \frac{\text{AIR}}{\text{AIR}_{\text{max}}})
$$

**参数**：
- α = 0.7 (准确性权重)
- AIR_max = 5 (最大可接受轮次)

**解释**：
- DA-Score ∈ [0, 1]
- 越高越好
- 平衡准确率和交互成本

#### 2.5.2 Efficiency-Accuracy Tradeoff

**定义**：准确率 - 轮次曲线下的面积。

$$
\text{EA-Tradeoff} = \int_{0}^{\text{AIR}_{\text{max}}} \text{Accuracy}(\text{rounds}) \, d(\text{rounds})
$$

**说明**：
- 曲线越靠近左上角越好
- 面积越大表示 tradeoff 越好

#### 2.5.3 User Experience Index

**定义**：综合体验指标的加权平均。

$$
\text{UEI} = 0.4 \times \text{Satisfaction} + 0.3 \times (1 - \text{Cognitive Load}) + 0.3 \times \text{Trust}
$$

### 3.2 Baseline 复现计划

#### 3.2.1 Odin (Personalizer + Conformal Prediction)

**核心方法**：
- 用户画像用于 Schema 选择
- 生成多个候选 SQL
- Conformal Prediction 筛选

**复现难点**：
- 用户画像表示不同
- Conformal Prediction 阈值调优

**适配方案**：
```python
class OdinBaseline:
    def __init__(self):
        self.user_embedder = UserEmbeddingModel()
        self.schema_selector = SchemaSelector()
        self.sql_generator = LLMGenerator()
        self.conformal_filter = ConformalFilter(alpha=0.1)
    
    def predict(self, query, user_id):
        # 1. 用户画像嵌入
        user_emb = self.user_embedder.encode(user_id)
        
        # 2. Schema 选择
        schema_candidates = self.schema_selector.select(query, user_emb, k=5)
        
        # 3. SQL 生成
        sql_candidates = []
        for schema in schema_candidates:
            sql = self.sql_generator.generate(query, schema)
            sql_candidates.append(sql)
        
        # 4. Conformal 筛选
        final_sqls = self.conformal_filter.filter(sql_candidates)
        
        return final_sqls
```

#### 3.2.2 LinkAlign (Multi-Agent Schema Linking)

**核心方法**：
- 多 Agent 辩论
- 多轮检索验证
- 一致性投票

**复现难点**：
- Agent 角色定义
- 辩论轮次控制

**适配方案**：
```python
class LinkAlignBaseline:
    def __init__(self):
        self.agents = {
            'retriever': RetrieverAgent(),
            'linker': LinkingAgent(),
            'validator': ValidationAgent()
        }
        self.debate_rounds = 3
    
    def predict(self, query):
        # 多轮辩论
        for round in range(self.debate_rounds):
            # 检索
            retrieved = self.agents['retriever'].retrieve(query)
            # 链接
            linked = self.agents['linker'].link(query, retrieved)
            # 验证
            validated = self.agents['validator'].validate(linked)
            
            if validated.consensus:
                break
        
        return linked.schema_mapping
```

#### 3.2.3 AmbiSQL (Interactive Disambiguation)

**核心方法**：
- 基于规则的歧义检测
- 模板化澄清问题
- 用户反馈更新

**复现难点**：
- 规则库构建
- 模板覆盖度

**适配方案**：
```python
class AmbiSQLBaseline:
    def __init__(self):
        self.ambiguity_rules = self.load_rules()
        self.clarification_templates = self.load_templates()
        self.max_rounds = 3
    
    def predict(self, query, context):
        for round in range(self.max_rounds):
            # 检测歧义
            ambiguities = self.detect_ambiguities(query, self.ambiguity_rules)
            
            if not ambiguities:
                return self.generate_sql(query)
            
            # 生成澄清问题
            question = self.generate_question(ambiguities[0], self.clarification_templates)
            
            # 用户反馈 (模拟)
            answer = self.simulate_user_answer(question)
            
            # 更新查询
            query = self.refine_query(query, answer)
        
        return self.generate_sql(query)
```

## 四、统计检验方法

### 4.1 显著性检验

**方法**：配对 t 检验 (Paired t-test)

**假设**：
- H₀: TriCo-DAG 与 Baseline 无显著差异
- H₁: TriCo-DAG 显著优于 Baseline

**显著性水平**：α = 0.05

**效应量**：Cohen's d

$$
d = \frac{\bar{X}_1 - \bar{X}_2}{s_{\text{pooled}}}
$$

| d 值 | 效应大小 |
|------|----------|
| 0.2 | 小效应 |
| 0.5 | 中效应 |
| 0.8 | 大效应 |

### 4.2 置信区间

**方法**：Bootstrap 重采样 (1000 次)

**置信水平**：95%

**报告格式**：
```
TriCo-DAG DA-Score: 0.82 [95% CI: 0.79 - 0.85]
Odin DA-Score:      0.71 [95% CI: 0.68 - 0.74]
p < 0.001, d = 0.67 (中到大效应)
```
