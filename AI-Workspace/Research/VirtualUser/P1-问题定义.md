## 一、Background（研究背景）

### 1.1 场景描述

**市场调研的真实场景**：

京东作为电商平台，每天都在做市场调研决策：
- 新品上市前：这款产品会受哪类用户欢迎？定价多少合适？
- 营销活动：618/双11 的促销策略对不同用户群体效果如何？
- 用户洞察：高价值用户有什么特征？流失用户为什么离开？
- 竞品分析：竞品用户和我们的用户有什么差异？

**典型工作流程**：
```
业务需求 → 问卷设计 → 招募用户 → 数据收集 → 分析报告 → 决策支持
              ↓            ↓
           1-2 周      2-4 周 + ¥5-50万
```

**关键问题**：
- **时间成本**：从需求到结论至少 1 个月
- **金钱成本**：单次调研 ¥5-50 万（用户招募、问卷投放、数据处理）
- **样本局限**：能触达的用户有限，代表性不足
- **迭代困难**：方案调整需要重新跑一遍流程

### 1.2 面临瓶颈

| 瓶颈 | 具体问题 | 业务影响 | 量化示例 |
|------|---------|---------|---------|
| **招募难** | 找到目标用户群体困难 | 样本偏差，结论不可靠 | 25-35岁一线城市女性，招募周期 2-3 周 |
| **成本高** | 每次调研都要花钱招募用户 | 预算有限，调研频次低 | 单次问卷成本 ¥50-100/人，样本 1000 人 = ¥5-10 万 |
| **周期长** | 从需求到结论周期长 | 错过市场窗口 | 平均周期 4-6 周 |
| **不可迭代** | 方案修改要重来 | 决策不灵活 | 调整问卷题目需要重新投放 |

**真实案例**：

**案例 1：新品定价调研**
> 某家电品牌新品上市，需要测试 3 个价位点的用户接受度。
> - 传统方法：招募 500 人，问卷投放 2 周，成本 ¥5 万
> - 问题：竞品已经抢先上市，错过最佳定价窗口

**案例 2：用户流失原因分析**
> 某品类用户流失率上升，需要了解原因。
> - 传统方法：电话回访流失用户，接通率 < 10%
> - 问题：能联系上的用户不代表真实流失原因

**案例 3：营销文案测试**
> 618 活动需要测试 5 套营销文案的效果。
> - 传统方法：A/B 测试需要真实流量，影响转化
> - 问题：测试期间损失真实订单

### 1.3 发展现状

**行业尝试**：

| 公司/产品 | 方案 | 效果 | 问题 |
|----------|------|------|------|
| **问卷星 AI 问卷** | AI 生成问卷题目 | ⭐⭐⭐ | 只解决问卷设计，不解决用户模拟 |
| **SurveyMonkey Genius** | AI 分析问卷结果 | ⭐⭐⭐ | 基于已有数据，不能替代用户 |
| **ChatGPT 模拟用户** | 直接问 LLM | ⭐⭐ | 响应质量不稳定，缺乏真实用户特征 |

**技术可行性验证**：

2023-2025 年学术界已有探索：
- LLM 可以模拟特定 Persona 的问卷回答（Polypersona 2025）
- Persona 条件化能提升模拟质量（LLM-S3 2025）
- 但态度一致性问题未解决（Lost in Simulation 2026）

**京东的独特优势**：
- ✅ **海量行为数据**：用户浏览、搜索、购买、评价数据
- ✅ **丰富用户画像**：人口统计、消费能力、品类偏好
- ✅ **真实调研需求**：业务方有明确的调研场景和预算
- ✅ **验证环境**：可以用真实业务数据验证虚拟用户模拟效果

## 二、What（研究目标）

### 2.1 Goal（定位与理念）

**产品定位**：
> **虚拟用户市场调研系统**：基于京东电商行为数据，构建虚拟用户，快速生成市场调研响应，帮助业务方低成本、高效率完成调研决策。

**核心理念**：
- **数据驱动**：用真实用户行为数据构建虚拟用户，而非凭空生成
- **快速迭代**：分钟级生成响应，支持方案快速调整
- **可信可控**：虚拟用户态度一致，响应可解释

**应用场景**：

| 场景         | 传统方法      | 虚拟用户方法     | 价值                    |
| ---------- | --------- | ---------- | --------------------- |
| **新品测试**   | 招募用户问卷调研  | 虚拟用户响应     | 时间从 4 周 → 1 天，成本降 90% |
| **定价研究**   | A/B 测试或问卷 | 虚拟用户定价偏好模拟 | 不影响真实流量，快速验证          |
| **用户分群**   | 聚类分析      | 构建各群体虚拟用户  | 深入理解群体特征和需求           |
| **营销文案测试** | 小流量测试     | 虚拟用户反馈     | 快速筛选最优方案              |

### 2.2 Results（预期成果）

#### 量化指标

**系统性能指标**：

| 指标         | 目标值              | 对比基准            |
| ---------- | ---------------- | --------------- |
| **响应生成速度** | 1000 虚拟用户 < 5 分钟 | 传统问卷 2-4 周      |
| **单次调研成本** | < ¥500（API 调用成本） | 传统问卷 ¥5-50 万    |
| **虚拟用户数量** | 支持 10万+ 虚拟用户     | 传统问卷样本 500-2000 |

**响应质量指标**：

| 指标        | 目标值   | 验证方法            |
| --------- | ----- | --------------- |
| **态度一致性** | > 85% | 同一虚拟用户对相关问题回答一致 |
| **分布相似度** | > 90% | 与真实用户调研结果分布对比   |
| **预测准确率** | > 80% | 预测真实用户行为（如购买转化） |

#### 产出物

| 产出物          | 描述           | 交付标准           |
| ------------ | ------------ | -------------- |
| **虚拟用户构建系统** | 基于电商数据生成虚拟用户 | 可生成指定特征的虚拟用户   |

| **评估验证工具**   | 验证虚拟用户响应质量   | 一致性/分布/预测三维度评估 |

## 三、Why（研究意义与挑战）

### 3.1 Values（价值分析）

#### 3.1.1 业务价值

**直接价值**：

| 价值维度 | 具体收益 | 量化估算 |
|---------|---------|---------|
| **成本节省** | 减少用户招募和问卷投放成本 | 年节省调研预算 50%+（预估 ¥500万+）|
| **效率提升** | 缩短调研周期，加速决策 | 周期从 4 周 → 1 天 |
| **样本扩展** | 支持更大样本量，提升统计显著性 | 样本量从 1000 → 10万+ |
| **敏捷迭代** | 快速调整方案，不增加成本 | 支持 10+ 轮迭代 |

**间接价值**：

| 价值维度 | 具体收益 |
|---------|---------|
| **决策质量提升** | 更快速、更低成本的试错，找到最优方案 |
| **创新探索** | 原本因成本放弃的调研想法可以快速验证 |
| **用户体验保护** | 减少对真实用户的打扰 |

#### 3.1.2 技术价值

| 价值维度 | 具体贡献 |
|---------|---------|
| **方法论创新** | 提出电商数据驱动的虚拟用户构建方法 |
| **态度一致性** | 解决 LLM 模拟的核心痛点——态度不一致 |
| **可复用性** | 方法可推广到其他电商平台、内容平台 |

#### 3.1.3 学术价值

| 价值维度 | 具体贡献 |
|---------|---------|
| **填补空白** | 首次在电商场景系统性研究虚拟用户市场调研 |
| **实证验证** | 真实业务数据验证方法论有效性 |
| **开源贡献** | 开源评估工具和数据集 |

### 3.2 Challenges（挑战分析）

| 挑战 | 描述 | 难度 | 解决思路 |
|------|------|------|---------|
| **C1: 虚拟用户构建** | 如何从电商行为数据构建真实可信的 Persona？ | 高 | 行为数据 → 态度推断 → Persona 生成 |
| **C2: 态度一致性** | 虚拟用户对相关问题的回答是否一致？ | 高 | 态度建模 + 一致性约束 |
| **C3: 响应真实性** | 虚拟响应与真实用户响应有多接近？ | 中 | 多维度评估 + 真实数据验证 |
| **C4: 业务有效性** | 虚拟用户能否帮助业务决策？ | 中 | 真实业务场景验证 |

#### C1: 虚拟用户构建

**问题**：
- 电商数据主要是行为（浏览、购买），如何推断用户态度？
- 如何构建完整、多样化的虚拟用户 Persona？
- Persona 如何与真实用户群体对齐？

**解决思路**：
- 行为数据态度推断模型
- 人口统计分布约束的 Persona 采样
- 基于用户分群的 Persona 构建

#### C2: 态度一致性

**问题**：
- 同一虚拟用户对"您喜欢户外运动吗？"和"您周末会去健身房吗？"应该回答一致
- 现有 LLM 方法存在态度矛盾（如回答喜欢运动但不愿付费办卡）
- 一致性是虚拟用户可信的核心

**解决思路**：
- 态度因子建模（消费态度、生活方式态度、品牌态度等）
- 态度一致性约束生成
- 后验证修正机制

#### C3: 响应真实性

**问题**：
- 如何验证虚拟用户响应接近真实用户？
- 没有完美 baseline（真实用户本身也有随机性）
- 需要多维度评估

**解决思路**：
- 分布相似度（与真实调研结果对比）
- 态度一致性（内部一致性）
- 预测有效性（能否预测真实行为）

#### C4: 业务有效性

**问题**：
- 学术验证 ≠ 业务有效
- 需要在真实场景证明价值
- 业务方接受度

**解决思路**：
- 与业务方合作设计验证实验
- A/B 对比（虚拟用户预测 vs 真实用户行为）
- 业务指标量化（成本节省、效率提升）

## 四、应用场景详细设计

### 场景 1：新品定价测试

**业务背景**：
某家电品牌新品上市，需要测试 3 个价位点的用户接受度。

**传统方法**：
1. 设计问卷，包含价格敏感度题目
2. 招募 500 目标用户
3. 投放问卷，收集 2 周
4. 分析结果，提交报告
5. 成本：¥5 万，周期：4 周

**虚拟用户方法**：
1. 从京东数据构建目标用户群体的虚拟用户（如 25-40 岁一线城市白领）
2. 虚拟用户填写问卷
3. 分析结果
4. 成本：¥500，周期：1 天

**验证方式**：
- 对比虚拟用户预测的价格接受度 vs 实际上市后的价格接受度

### 场景 2：用户流失原因分析

**业务背景**：
某品类用户流失率上升，需要了解原因。

**传统方法**：
1. 电话回访流失用户，接通率低
2. 问卷投放，响应率低
3. 能联系到的用户可能有偏差

**虚拟用户方法**：
1. 基于流失用户的行为数据构建虚拟流失用户
2. 虚拟用户回答流失原因问卷
3. 对比留存用户的虚拟用户，分析差异

**验证方式**：
- 虚拟用户识别的流失原因 vs 实际客服记录的流失原因

### 场景 3：营销文案筛选

**业务背景**：
618 活动准备 5 套营销文案，需要筛选最优方案。

**传统方法**：
1. 小流量 A/B 测试
2. 需要真实流量，可能影响转化
3. 测试周期 1 周

**虚拟用户方法**：
1. 构建目标用户群体的虚拟用户
2. 虚拟用户对 5 套文案评分和反馈
3. 筛选出最优方案

**验证方式**：
- 虚拟用户偏好的文案 vs 真实 A/B 测试结果
