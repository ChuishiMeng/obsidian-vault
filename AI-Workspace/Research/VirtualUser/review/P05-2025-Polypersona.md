## 一、研究背景

### 1.1 问题场景（一句话说清楚）

**核心问题**：如何让 AI 模拟不同背景的人填写问卷，而且回答要真实、多样、不矛盾？

**通俗理解**：

想象你要做一个市场调研，传统做法是：
1. 招募 1000 个真实用户
2. 发问卷让他们填写
3. 等几周收集数据
4. 花费 5-50 万

现在想用 AI 来模拟：
1. 定义 1000 个虚拟用户（年龄、性别、职业等）
2. 让 AI 帮他们填问卷
3. 成本降到几百块，时间缩短到几小时

**问题来了**：AI 填的问卷靠谱吗？

```
[具体示例]

场景：做新品定价调研
传统做法：
  招募用户 → 发问卷 → 等反馈 → 分析数据
  成本：5-50万
  时间：4周

AI 模拟：
  定义虚拟用户 → AI 填问卷 → 分析数据
  成本：500元
  时间：1天

问题：
  AI 填的问卷真实吗？
  AI 会"精神分裂"吗？（前面说喜欢运动，后面说从不运动）
  AI 会"从众"吗？（所有人都回答相似的答案）
```

**传统方法的问题**：

| 传统方法 | 怎么做 | 问题 |
|---------|--------|------|
| **无 Persona 约束** | 直接问 AI "帮我填问卷" | AI 倾向给"友好"答案，缺乏多样性 |
| **简单 Persona** | 只给 AI 年龄、性别 | AI 缺乏背景，回答不一致 |
| **大模型直接用** | 用 GPT-4 直接生成 | 成本高，无法定制 |

### 1.2 为什么这个问题重要？

| 场景 | 为什么重要 |
|------|-----------|
| **学术** | 问卷调查是社会科学研究的核心方法，但成本高、周期长 |
| **企业** | 市场调研、用户研究、新品测试都需要大量问卷数据 |

### 1.3 核心挑战

| 挑战 | 描述 | 现有方案局限 |
|------|------|-------------|
| **一致性** | 同一个虚拟用户，不同问题回答要一致 | 无约束时 AI 容易"精神分裂" |
| **多样性** | 不同虚拟用户要有不同观点 | AI 倾向"从众"，给相似答案 |
| **真实性** | 回答要符合真实人类行为 | 缺乏大规模验证 |

## 二、核心贡献

### 2.1 一句话说清楚

**用 Persona（人口统计+心理特征）约束 AI，让小模型通过高效微调，生成真实多样的问卷响应。**

### 2.2 具体做了什么？

**思路**：构建 Persona-Question-Response 三元组数据集，微调小模型学习如何根据 Persona 填问卷。

**核心工作**：

**1. 构建 Polypersona 数据集**

```
传统方法：
  无结构化数据，直接用 AI 生成

本文方法：
  PersonaHub (1B personas) → 筛选 433 个
  QuestionBank (成熟问卷) → 82 个问题
  配对生成 → 3,568 个响应
```

**2. 设计 LoRA 微调方案**

| 步骤 | 做什么 | 为什么 |
|------|--------|--------|
| 选择模型 | TinyLlama 1.1B | 小模型，消费级 GPU 可训练 |
| LoRA 配置 | rank=16, alpha=32 | 参数高效微调，只训练 2% 参数 |
| 4-bit 量化 | 减少内存 65% | 支持消费级 GPU |

### 2.3 核心创新

| 创新 | 描述 | 效果 |
|------|------|------|
| **方法创新** | Persona-grounded 数据管道 | 结构化约束，保证一致性 |
| **技术创新** | LoRA + 4-bit 微调小模型 | 消费级 GPU 可训练，成本低 |
| **资源创新** | 开源 QuestionBank | 基于成熟问卷，可直接复用 |

## 三、方法论

### 3.1 整体思路（类比理解）

**把 AI 训练成一个"角色扮演高手"**：

```
输入 → 处理流程 → 输出

Persona（角色卡）→ AI 扮演 → 问卷回答

类比：
  演员（AI）+ 角色卡（Persona）+ 剧本（问卷）= 表演（回答）
  
关键：演员要"入戏"，不同角色要有不同表现
```

### 3.2 分步流程

**第一步：构建 Persona**

```
做什么：定义虚拟用户的人口统计和心理特征
为什么：约束 AI 行为，保证一致性
怎么做：
  1. 从 PersonaHub 筛选 personas
  2. 结构化为 persona cards：
     - 人口统计：年龄、性别、职业
     - 心理特征：兴趣、价值观
     - 行为模式：消费习惯、生活方式
```

**第二步：构建 QuestionBank**

```
做什么：收集成熟问卷的题目
为什么：保证题目质量，避免随意设计
怎么做：
  1. 来源：HCAHPS, NSSE, TAM, NEP, Gallup Q12 等
  2. 分类：10 个领域（医疗、教育、消费等）
  3. 格式：开放题、Likert、是/否、同意陈述
```

**第三步：生成训练数据**

```
做什么：Persona-Question 配对，生成响应
为什么：让 AI 学习如何根据 Persona 回答问题
怎么做：
  1. 配对策略：每个 Persona 回答多个问题
  2. 格式：ChatML（system/user/assistant）
  3. 规模：3,568 个响应
```

**第四步：LoRA 微调**

```
做什么：高效微调小模型
为什么：成本低，效果好
怎么做：
  1. 模型：TinyLlama 1.1B
  2. LoRA 配置：rank=16, alpha=32
  3. 量化：4-bit
  4. 训练：3 epochs
```

### 3.3 具体例子

**场景**：定义一个 35 岁女性教师，让她回答关于"线上教育"的问卷

```
┌─────────────────────────────────────────────┐
│ 第一步：Persona 定义                          │
│   年龄：35岁                                  │
│   性别：女                                    │
│   职业：高中教师                               │
│   特征：关注教育公平，喜欢尝试新技术            │
├─────────────────────────────────────────────┤
│ 第二步：问卷题目                              │
│   "您认为线上教育能否替代线下教育？"            │
├─────────────────────────────────────────────┤
│ 第三步：AI 生成回答                           │
│   "作为一名教师，我认为线上教育可以作为补充，   │
│    但无法完全替代线下教育。线上教育在灵活性    │
│    方面有优势，但缺乏师生互动和情感交流..."    │
├─────────────────────────────────────────────┤
│ 第四步：一致性检验                            │
│   同一 Persona 回答相关问题，检查是否矛盾      │
└─────────────────────────────────────────────┘
```

### 3.4 技术细节

| 组件 | 实现方式 | 备注 |
|------|---------|------|
| **模型** | TinyLlama 1.1B | 小模型，消费级 GPU |
| **LoRA 目标层** | attention + MLP | 全覆盖效果好 |
| **rank** | 16 | 性能-效率平衡点 |
| **量化** | 4-bit | 减少内存 65% |
| **训练轮数** | 3 | 避免过拟合 |

## 四、实验结果

### 4.1 实验设置

**数据集**：

| 数据集 | 规模 | 用途 |
|--------|------|------|
| Polypersona | 3,568 响应 / 433 personas / 82 questions | 训练 + 测试 |

**Baseline**：

| Baseline | 描述 |
|----------|------|
| Phi-2 | 2.7B 参数 |
| Mistral 7B | 7B 参数 |
| DeepSeek R1 Distill | 1.5B 参数 |
| Qwen2 1.5B | 1.5B 参数 |

### 4.2 主实验结果

| 方法 | BLEU | ROUGE-L | BERT-F1 | Survey Quality |
|------|-------|--------|---------|----------------|
| **PolyPersona (1.1B)** | **0.090** | **0.239** | 0.890 | 0.873 |
| Mistral 7B | 0.085 | 0.228 | 0.887 | 0.852 |
| Phi-2 | 0.087 | 0.234 | **0.891** | 0.868 |

**关键发现**：1.1B 小模型媲美 7B 大模型！

### 4.3 消融实验

| 变体 | BLEU | 变化 |
|------|------|------|
| 完整方法 | 0.090 | - |
| w/o LoRA (仅 attention) | 0.082 | ↓8.9% |
| w/o 4-bit 量化 | - | 内存增加 65% |

### 4.4 关键发现

1. **发现1**：小模型 + LoRA 可以达到大模型性能
2. **发现2**：Persona 约束有效，跨领域 Persona 有一致性优势
3. **发现3**：多维度评估必要，单一指标不足
