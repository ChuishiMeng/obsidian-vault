## 一、研究背景

### 1.1 问题场景

**场景描述**：用户问"纽约哪些社区深夜签到最多"，需要时空推理。但数据库只有坐标，没有社区字段。

**具体示例**：
```
用户问题："纽约哪些社区在深夜签到最多？"

数据库字段：latitude, longitude, checkin_time
缺失信息：社区边界定义、深夜时间范围

挑战：
1. "社区"不在 schema 中
2. "深夜"需要时间推理
3. 需要外部知识注入
```

### 1.2 学术场景 vs 企业场景对比

| 维度 | 学术基准 | 企业实际 |
|------|---------|---------|
| Schema 完整性 | 字段齐全 | 缺少业务字段 |
| 外部知识 | 不需要 | 需要领域知识 |
| 可视化 | 无要求 | 地图/图表输出 |

## 二、核心贡献

### 2.1 主要贡献

1. **ReAct Agent 架构**：Plan-Act-Observe 循环解决复杂问题
2. **六大工具协作**：Schema 检查、SQL 生成、执行验证、地图、图表
3. **准确率大幅提升**：从 28.6% 到 91.4%

### 2.2 创新点

- **方法创新**：Agent 架构替代单次生成
- **技术创新**：多工具协作解决时空推理
- **应用创新**：可视化输出增强可用性

## 三、方法论

### 3.1 整体框架

```
Naive Pipeline：Question → SQL → Execute → 28.6% 准确率

Agentic Pipeline：
Question → ReAct Agent
              ↓
         Plan → Act → Observe
              ↓
         六大工具协作
              ↓
         91.4% 准确率
```

### 3.2 核心方法

#### 方法1：ReAct 循环

**输入**：用户问题
**输出**：SQL + 结果 + 可视化
**核心步骤**：
1. Plan：规划下一步
2. Act：执行工具
3. Observe：更新上下文
4. 循环直到完成

**关键示例**：
```
Question: "纽约哪些社区深夜签到最多？"

[Plan 1] 理解"深夜"和"社区"
[Act 1] get_database_schema_tool → 查看字段
[Observe 1] 有 latitude, longitude, checkin_time

[Plan 2] "社区"字段缺失，需要外部知识
[Act 2] mapping_tool → 获取社区边界
[Observe 2] 已获取社区划分数据

[Plan 3] 构建深夜签到 SQL
[Act 3] generate_sql_query_tool
[Observe 3] SQL 生成完成

[Plan 4] 执行并可视化
[Act 4] execute + mapping_tool
[Observe 4] 热力图生成完成
```

#### 方法2：六大工具协作

| 工具 | 功能 | 关键作用 |
|------|------|---------|
| `get_database_schema_tool` | 获取 schema | Schema 感知 |
| `generate_sql_query_tool` | 生成 SQL | 核心生成 |
| `execute_on_database_tool` | 执行验证 | 错误恢复 |
| `mapping_tool` | 地图可视化 | 空间推理 |
| `plotting_tool` | 图表生成 | 趋势分析 |

### 3.3 技术细节

| 组件 | 实现方式 | 备注 |
|------|---------|------|
| 基础 LLM | GPT-4 | 推理核心 |
| 工具调用 | Function Calling | 标准接口 |
| 可视化 | Matplotlib + Folium | 地图/图表 |

## 四、实验结果

### 4.1 实验设置

**数据集**：

| 数据集 | 规模 | 用途 |
|--------|------|------|
| 时空查询 | 多源 | 主实验 |

### 4.2 主实验结果

| 查询类型 | Naive | Agentic | 提升 |
|---------|-------|---------|------|
| 空间查询 | 25.7% | **88.6%** | +62.9% |
| 时间查询 | 31.4% | **94.3%** | +62.9% |
| 多数据集 | 28.6% | **91.4%** | +62.8% |

### 4.3 关键发现

1. **Agent 架构大幅提升准确率**：+62.8%
2. **多工具协作有效**：解决外部知识缺失问题
3. **可视化增强可用性**：用户可直接使用结果
