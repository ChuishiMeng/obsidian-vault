## 基本信息

- **论文标题**: CHESS: Contextual Harnessing for Efficient SQL Synthesis
- **发表时间**: 2024年5月（更新版2024年11月）
- **核心方向**: 四Agent多Agent框架，开源模型SOTA，工业级部署

## 核心问题

**如何在工业级大规模Schema场景下，用开源模型实现接近闭源大模型的效果？**

CHESS 通过四个专业Agent协同解决 Schema复杂度、查询规划和代码质量验证。

## 技术方法

### 四Agent架构

```
用户问题
    ↓
┌─────────────────────────────┐
│ Agent 1: Information Retriever (IR) │
│ 提取外部知识（文档、知识库）        │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│ Agent 2: Schema Selector (SS) │ ⭐ 核心
│ 渐进式Schema剪枝：96列 → ~10列  │
│ 提升准确率 ~2%，减少 Token ×5     │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│ Agent 3: Code Generator (CG)  │
│ 生成SQL                        │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│ Agent 4: Unit Tester (UT)     │
│ 执行验证 + 自动修订              │
└─────────────────────────────┘
    ↓
最终SQL
```

### 关键特性
1. **工业级Schema支持**：Schema Selector 处理超大规模数据库
2. **隐私保护**：完全基于开源模型，可在私有环境部署
3. **可配置**：支持不同计算预算的部署模式
4. **模块化**：每个Agent可独立替换/升级

## 与 DA-CaseBase 的关系

### ⭐ 最重要的 Baseline

CHESS 是 BIRD 基准上**开源模型的天花板**，DA-CaseBase 的核心对比对象。

### 可借鉴的技术点

1. **渐进式Schema剪枝** → 案例库可以预存 Schema 子集，直接跳过剪枝步骤
2. **Unit Tester 验证机制** → 可直接迁移到案例库的自动质量验证
3. **模块化设计** → 案例库可以作为新Agent插入 CHESS 框架

### 架构对比

| 维度 | CHESS | DA-CaseBase |
|------|-------|-------------|
| Agent数量 | 4个 | 3层（缓存→模板→生成）|
| Schema处理 | 在线剪枝 | 案例预存Schema |
| LLM调用 | 少量（开源模型）| 命中时0调用 |
| 知识来源 | 外部文档 | 历史案例 |
| 适用场景 | 通用 | 特定领域高频查询 |

### 本研究的切入点
- CHESS 的 Schema Selector 需要在线推理 → DA-CaseBase 的案例匹配是**离线预计算**
- CHESS 是"每次都推理" → DA-CaseBase 是"匹配过就缓存"
- **组合潜力**：CaseBase + CHESS = 先匹配案例，未命中再用CHESS生成
