## 四、核心论文解读

### 4.1 五星论文（本研究核心方法）

#### P02-2026-CBR-to-SQL - 案例驱动的 Text-to-SQL ⭐⭐⭐⭐⭐

**一句话说清楚**：用"模板 + 实体"两阶段分离，实现 SQL 案例的泛化复用。

**核心方法**：

```
两阶段分离：

Step 1: 模板构建（离线）
  问题："哮喘患者用过布地奈德"
  SQL: SELECT COUNT(*) FROM diagnoses WHERE icd9='493' AND drug='Budesonide'
  模板: SELECT COUNT(*) FROM {table} WHERE {disease_col}={disease_val} AND {drug_col}={drug_val}

Step 2: 实体落地（在线）
  新问题："糖尿病用过二甲双胍"
  模板匹配 + 实体映射 → 最终 SQL
```

**效果**：Acc_LF 82.8%（医疗场景）

**借鉴价值**：✅ 本研究的核心技术路线

#### P15-2026-Semantic-Cache-OLAP - LLM 查询规范化

**一句话说清楚**：用 LLM 规范化查询语句，命中率从 28% 提升到 76%。

**核心方法**：
```
传统方法：直接 Embedding 匹配
  问题："销售额" vs "销售总额" → 不匹配

LLM 规范化：
  "销售额" → "SELECT SUM(amount) FROM sales"
  "销售总额" → "SELECT SUM(amount) FROM sales"
  → 规范化后匹配 → 命中！
```

**借鉴价值**：✅ Layer 1 语义缓存的核心技术

---

#### P33-2025-SQL-Equivalence - SQL 等价性评估

**一句话说清楚**：用 LLM 判断两个 SQL 是否等价，准确率 92.3%。

**核心方法**：
```
传统指标问题：
  - Execution Accuracy：假阳性（空结果匹配）
  - Exact Set Matching：假阴性（格式差异）

Miniature + Mull 策略：
  1. Miniature：构造边界值测试数据
  2. Mull：多角度质疑等价性
  3. 多次运行投票
```

**借鉴价值**：✅ 案例匹配时的等价性判断

---

#### P34-2025-HES-SQL - 混合推理框架

**一句话说清楚**：既"对"又"快"：简单查询直接生成，复杂查询推理生成。

**核心方法**：
```
自适应推理模式：
  简单查询（1-2表）→ 直接生成 → 快
  复杂查询（多表）→ CoT 推理 → 准

骨架完整性评分：
  提取 SQL 骨架 = {SELECT, JOIN, WHERE, GROUP BY, ORDER BY}
  与最优骨架对比，计算对齐度
```

**借鉴价值**：✅ SQL 骨架提取、复杂度分级
