## 核心用户故事

### US-01：一句话生成页面结构

#### 基本信息

| 项目       | 内容            |
| -------- | ------------- |
| **角色**   | 运营人员、产品人员     |
| **优先级**  | P0 - 核心功能     |
| **故事点**  | 8             |
| **关联功能** | 文字生成页面、模板智能推荐 |

#### 用户故事

```
作为一个 运营人员/产品人员，
我希望 通过一句话描述就能生成完整的页面结构，
以便 在5分钟内完成页面骨架搭建，节省80%的配置时间。
```

#### 验收标准

**AC-01：基础生成**

```gherkin
Given 用户在乐高页面运营界面
When 用户点击"AI创建"按钮，输入"创建一个双11活动页面，包含banner、优惠券、秒杀楼层"
Then 系统在5秒内返回页面结构预览
And 生成的楼层与用户意图匹配度 >= 80%
And 每个楼层都匹配到合适的模板
And 用户可以一键将结构应用到画布
```

**AC-02：多方案选择**

```gherkin
Given 用户输入了页面描述
When AI生成完成
Then 系统展示2-3个不同风格的页面方案
And 每个方案都可以预览
And 用户可以选择其中一个应用到画布
```

**AC-03：意图澄清**

```gherkin
Given 用户输入的描述不够明确（如"帮我做一个页面"）
When 系统无法准确理解意图
Then 系统主动提问澄清："请问您想创建什么类型的页面？频道页/活动页/详情页？"
And 提供选项供用户快速选择
And 用户选择后系统继续生成
```

**AC-04：对话式调整**

```gherkin
Given 用户已经生成了页面结构
When 用户输入"把优惠券楼层放到最上面"
Then 系统调整楼层顺序
And 画布实时更新
And 用户可以继续用对话调整其他内容
```

**AC-05：异常处理**

```gherkin
Given 用户输入包含不存在的模板需求
When 系统检测到无法满足的请求
Then 系统提示"暂未找到XX类型的模板，是否使用相似模板替代？"
And 展示可替代的模板选项
And 用户确认后继续
```

#### 技术备注

- 意图理解：LLM + Few-shot
- 模板匹配：向量检索 + 规则过滤
- 响应时间：< 5秒
- 准确率目标：>= 80%

### US-02：智能生成坑位文案

#### 基本信息

| 项目       | 内容        |
| -------- | --------- |
| **角色**   | 运营人员      |
| **优先级**  | P0 - 核心功能 |
| **故事点**  | 5         |
| **关联功能** | 素材文案生成    |

#### 用户故事

```
作为一个 运营人员，
我希望 AI能根据坑位类型和页面主题自动生成多个文案选项，
以便 快速完成素材文案配置，减少文案创作时间。
```

#### 验收标准

**AC-01：基础生成**

```gherkin
Given 用户选中一个文案类坑位（标题/按钮/描述）
When 用户右键选择"AI生成文案"
Then 系统根据坑位类型、页面主题生成3-5个文案选项
And 每个文案符合品牌调性
And 用户可以一键选择应用
```

**AC-02：风格控制**

```gherkin
Given 用户需要生成特定风格的文案
When 用户在生成时选择风格（活泼/专业/温馨/促销）
Then 生成的文案符合选定风格
And 用户可以重新生成不同风格
```

**AC-03：字数限制**

```gherkin
Given 坑位有字数限制
When 用户生成文案
Then 所有文案选项都符合字数限制
And 超出限制的文案自动过滤
```

**AC-04：历史学习**

```gherkin
Given 用户多次生成同类坑位文案
When 用户选择并应用某个文案
Then 系统记录用户偏好
And 后续生成时优先推荐相似风格文案
```

**AC-05：批量生成**

```gherkin
Given 用户选中多个同类坑位
When 用户选择"批量生成文案"
Then 系统为所有选中坑位生成文案
And 每个坑位的文案主题一致但内容不同
And 用户可以逐个调整
```

#### 技术备注

- 文案生成：LLM + Prompt工程
- 风格控制：Prompt模板
- 响应时间：< 3秒
- 质量目标：用户采纳率 >= 60%

### US-03：模板智能推荐

#### 基本信息

| 项目 | 内容 |
|------|------|
| **角色** | 产品人员、运营人员 |
| **优先级** | P0 - 核心功能 |
| **故事点** | 5 |
| **关联功能** | 模板智能推荐 |

#### 用户故事

```
作为一个 产品人员/运营人员，
我希望 系统能根据我的描述推荐合适的楼层模板，
以便 快速找到需要的模板，减少搜索和试错时间。
```

#### 验收标准

**AC-01：描述搜索**

```gherkin
Given 用户在模板面板搜索框
When 用户输入"优惠券楼层"或"展示多张优惠券"
Then 系统返回匹配的模板列表
And 按相关度排序
And 显示匹配度分数和推荐原因
```

**AC-02：上下文推荐**

```gherkin
Given 用户已经在画布上添加了2个楼层
When 用户打开模板面板
Then 系统根据已有楼层推荐后续楼层
And 推荐结果显示"推荐"标签
And 用户可以一键添加
```

**AC-03：相似模板推荐**

```gherkin
Given 用户正在查看某个模板详情
When 用户点击"查看相似模板"
Then 系统返回风格、结构相似的模板
And 用户可以对比选择
```

**AC-04：模板预览**

```gherkin
Given 用户在模板列表中
When 用户鼠标悬停在模板卡片上
Then 显示模板预览图
And 显示模板的关键信息（坑位数量、适用场景）
And 用户可以快速拖拽到画布
```

**AC-05：模板筛选**

```gherkin
Given 用户需要特定类型的模板
When 用户使用筛选条件（分类、标签、状态）
Then 系统返回符合条件的模板
And 推荐结果按智能排序+筛选条件综合排序
```

#### 技术备注

- 向量化：BGE-M3
- 检索：Milvus
- 响应时间：< 1秒
- 准确率目标：Top 3命中率 >= 70%

### US-04：页面合规智能检查

#### 基本信息

| 项目       | 内容        |
| -------- | --------- |
| **角色**   | 运营人员、审批人员 |
| **优先级**  | P1 - 重要功能 |
| **故事点**  | 3         |
| **关联功能** | 页面合规检查    |

#### 用户故事

```
作为一个 运营人员/审批人员，
我希望 系统能自动检查页面配置是否完整、合规，
以便 在发布前发现问题，避免线上故障。
```

#### 验收标准

**AC-01：一键检查**

```gherkin
Given 用户完成了页面配置
When 用户点击"检查页面"按钮
Then 系统扫描所有楼层、坑位、素材
And 3秒内输出检查结果
And 按严重程度分类展示问题
```

**AC-02：问题分类**

```gherkin
Given 系统完成页面检查
When 检查发现问题
Then 问题分为"错误"和"警告"两类
And "错误"级别问题必须修复才能发布
And "警告"级别问题提示但不阻止发布
```

**AC-03：快速定位**

```gherkin
Given 检查结果中列出了问题
When 用户点击某个问题
Then 画布自动定位到对应的楼层/坑位
And 高亮显示问题元素
And 用户可以直接修复
```

**AC-04：检查项列表**

| 检查项 | 级别 | 说明 |
|--------|------|------|
| 必填素材缺失 | 错误 | 必填坑位未配置素材 |
| 图片尺寸不符 | 警告 | 图片尺寸不符合坑位要求 |
| 链接失效 | 错误 | 素材链接无法访问 |
| 策略时间冲突 | 错误 | 同一页面策略时间重叠 |
| 敏感词检测 | 错误 | 文案包含敏感词 |

**AC-05：批量修复**

```gherkin
Given 检查发现多个同类问题
When 用户选择"批量修复"
Then 系统提供批量处理方案
And 用户确认后自动修复
And 修复完成后重新检查
```

#### 技术备注

- 检查引擎：规则引擎
- 响应时间：< 3秒
- 覆盖率目标：>= 90%的常见问题

### US-05：对话式页面调整

#### 基本信息

| 项目 | 内容 |
|------|------|
| **角色** | 运营人员、产品人员 |
| **优先级** | P1 - 重要功能 |
| **故事点** | 5 |
| **关联功能** | 文字生成页面（迭代能力） |

#### 用户故事

```
作为一个 运营人员/产品人员，
我希望 通过对话方式调整已生成的页面，
以便 不需要手动操作就能完成细节优化。
```

#### 验收标准

**AC-01：楼层调整**

```gherkin
Given 用户已生成页面结构
When 用户输入"把秒杀楼层移到优惠券上面"
Then 系统调整楼层顺序
And 画布实时更新
And 显示调整后的预览
```

**AC-02：楼层删除**

```gherkin
Given 用户已生成页面结构
When 用户输入"删除banner楼层"
Then 系统确认后删除该楼层
And 其他楼层顺序自动调整
```

**AC-03：楼层替换**

```gherkin
Given 用户已生成页面结构
When 用户输入"把第一个楼层换成另一个优惠券模板"
Then 系统推荐相似的优惠券模板
And 用户选择后替换
```

**AC-04：内容修改**

```gherkin
Given 用户需要修改某个坑位内容
When 用户输入"把第一个优惠券的标题改成满100减20"
Then 系统定位到对应坑位
And 修改标题内容
And 显示修改结果
```

**AC-05：撤销操作**

```gherkin
Given 用户通过对话完成了多次调整
When 用户输入"撤销上一步"或"恢复到3步前"
Then 系统回退到指定状态
And 显示回退后的页面
```

#### 技术备注

- 对话状态管理：状态机
- 指令解析：LLM
- 撤销支持：操作历史栈
- 响应时间：< 2秒

## 非功能性需求

### NFR-01：性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 页面生成响应时间 | < 5秒 | 从用户输入到返回结果 |
| 文案生成响应时间 | < 3秒 | 单个坑位文案生成 |
| 模板搜索响应时间 | < 1秒 | 向量检索 |
| 页面检查响应时间 | < 3秒 | 全页面扫描 |

### NFR-02：准确率要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 意图理解准确率 | >= 80% | 用户描述正确解析为结构 |
| 模板匹配准确率 | >= 70% | Top 3命中率 |
| 文案用户采纳率 | >= 60% | 用户选择使用生成的文案 |

### NFR-03：可用性要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 系统可用性 | >= 99.5% | AI服务可用性 |
| 错误恢复时间 | < 30秒 | AI服务异常后恢复 |
| 降级策略 | 支持 | AI不可用时回退到人工 |

### NFR-04：安全性要求

| 要求 | 说明 |
|------|------|
| 数据不外传 | 用户数据在私有环境处理 |
| 内容审核 | 生成内容需通过敏感词检测 |
| 操作审计 | 所有AI操作记录日志 |
