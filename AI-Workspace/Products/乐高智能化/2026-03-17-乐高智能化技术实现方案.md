### 1.2 核心设计思想

**关键创新点：组件控制协议（CCP）**

LLM不直接操作乐高系统，而是输出标准化的"操作指令"，由执行引擎翻译为乐高API调用。

```
LLM输出 → CCP指令 → 执行引擎 → 乐高API → 页面渲染
```

## 三、组件控制协议（CCP）设计

### 3.1 CCP指令格式

```typescript
// CCP指令基础结构
interface CCPInstruction {
  instruction_id: string;      // 指令唯一ID
  instruction_type: 'create' | 'update' | 'delete' | 'query';
  target_type: 'page' | 'floor' | 'slot' | 'material';
  target_id?: string;          // 目标组件ID（update/delete时必需）
  payload: object;             // 操作内容
  context?: {
    page_id?: string;
    floor_id?: string;
    slot_id?: string;
  };
}
```

### 3.2 CCP指令类型详解

#### 创建指令（CREATE）

```json
{
  "instruction_id": "inst_001",
  "instruction_type": "create",
  "target_type": "floor",
  "payload": {
    "floor_name": "优惠券楼层",
    "template_id": "TPL_20260317001",
    "priority": 1,
    "slots": [
      {
        "slot_name": "标题",
        "slot_type": "文案",
        "materials": [
          {"content": {"text": "限时特惠"}}
        ]
      },
      {
        "slot_name": "优惠券1",
        "slot_type": "优惠券",
        "materials": [
          {"content": {"coupon_id": "CPN_001"}}
        ]
      }
    ]
  },
  "context": {
    "page_id": "PAGE_001"
  }
}
```

#### 更新指令（UPDATE）

```json
{
  "instruction_id": "inst_002",
  "instruction_type": "update",
  "target_type": "slot",
  "target_id": "slot_title_001",
  "payload": {
    "materials": [
      {"content": {"text": "更新后的标题"}}
    ]
  }
}
```

#### 删除指令（DELETE）

```json
{
  "instruction_id": "inst_003",
  "instruction_type": "delete",
  "target_type": "floor",
  "target_id": "floor_002"
}
```

### 3.3 CCP执行流程

```
1. LLM生成CCP指令（JSON格式）
2. 指令验证引擎校验格式和业务规则
3. 指令翻译器转换为乐高API调用
4. 执行结果返回给用户
5. 预览画布实时更新
```

### 5.2 API翻译器实现

```python
class APITranslator:
    """将CCP指令翻译为乐高API调用"""
    
    def translate(self, instruction: CCPInstruction) -> APIRequest:
        """翻译单条指令"""
        
        if instruction.target_type == 'floor':
            return self._translate_floor_instruction(instruction)
        elif instruction.target_type == 'slot':
            return self._translate_slot_instruction(instruction)
        elif instruction.target_type == 'material':
            return self._translate_material_instruction(instruction)
    
    def _translate_floor_instruction(self, instruction: CCPInstruction) -> APIRequest:
        """翻译楼层相关指令"""
        
        if instruction.instruction_type == 'create':
            return APIRequest(
                endpoint='/api/floor/create',
                method='POST',
                body={
                    'page_id': instruction.context['page_id'],
                    'template_id': instruction.payload['template_id'],
                    'floor_name': instruction.payload['floor_name'],
                    'priority': instruction.payload['priority']
                }
            )
        
        elif instruction.instruction_type == 'update':
            return APIRequest(
                endpoint=f'/api/floor/{instruction.target_id}',
                method='PUT',
                body=instruction.payload
            )
        
        elif instruction.instruction_type == 'delete':
            return APIRequest(
                endpoint=f'/api/floor/{instruction.target_id}',
                method='DELETE'
            )
```

### 5.3 规则校验器

```python
class RuleValidator:
    """校验CCP指令是否符合业务规则"""
    
    def validate(self, instructions: List[CCPInstruction]) -> ValidationResult:
        """校验指令列表"""
        
        errors = []
        page_state = self._get_current_page_state()
        
        for inst in instructions:
            # 校验模板是否存在
            if inst.target_type == 'floor' and inst.instruction_type == 'create':
                if not self._template_exists(inst.payload['template_id']):
                    errors.append(f"模板 {inst.payload['template_id']} 不存在")
            
            # 校验楼层数量限制
            if inst.instruction_type == 'create' and inst.target_type == 'floor':
                if page_state.floor_count >= 20:
                    errors.append("楼层数量已达上限20个")
            
            # 校验坑位类型匹配
            if inst.target_type == 'material':
                slot = self._get_slot(inst.context['slot_id'])
                material_type = inst.payload['material_type']
                if not self._type_match(slot.slot_type, material_type):
                    errors.append(f"素材类型 {material_type} 与坑位类型 {slot.slot_type} 不匹配")
        
        return ValidationResult(
            valid=len(errors) == 0,
            errors=errors
        )
```

## 六、模板智能匹配

### 6.1 向量化模板库

```python
class TemplateVectorStore:
    """模板向量存储"""
    
    def __init__(self):
        self.embeddings = {}  # template_id -> embedding
        self.metadata = {}    # template_id -> metadata
    
    def index_template(self, template: Template):
        """索引模板"""
        # 构建模板描述文本
        description = self._build_description(template)
        
        # 生成向量
        embedding = self.embedding_model.encode(description)
        
        # 存储
        self.embeddings[template.template_id] = embedding
        self.metadata[template.template_id] = {
            'template_id': template.template_id,
            'template_name': template.template_name,
            'category': template.category,
            'tags': template.tags,
            'description': description
        }
    
    def _build_description(self, template: Template) -> str:
        """构建模板描述文本"""
        parts = [
            f"模板名称：{template.template_name}",
            f"分类：{template.category}",
            f"标签：{', '.join(template.tags)}",
            f"描述：{template.description}",
            f"适用场景：{', '.join(template.applicable_scenarios)}",
            f"坑位信息：{self._describe_slots(template.slots)}"
        ]
        return '\n'.join(parts)
    
    def search(self, query: str, top_k: int = 5) -> List[TemplateMatch]:
        """检索相似模板"""
        query_embedding = self.embedding_model.encode(query)
        
        scores = {}
        for template_id, embedding in self.embeddings.items():
            score = cosine_similarity(query_embedding, embedding)
            scores[template_id] = score
        
        # 排序返回TopK
        sorted_templates = sorted(scores.items(), key=lambda x: x[1], reverse=True)
        return [
            TemplateMatch(
                template_id=tid,
                score=score,
                metadata=self.metadata[tid]
            )
            for tid, score in sorted_templates[:top_k]
        ]
```

### 6.2 模板匹配流程

```
用户输入："我需要一个优惠券楼层"

        ↓

        ↓
```

## 十、总结

### 核心技术要点

| 要点 | 说明 |
|------|------|
| **组件描述Schema** | 让LLM"理解"乐高的组件体系 |
| **CCP协议** | 标准化LLM输出，解耦AI与业务系统 |
| **向量检索** | 实现模板智能匹配 |
| **规则校验** | 保证生成结果符合业务约束 |
| **对话状态管理** | 支持多轮交互 |

### 关键优势

1. **可扩展**：新增模板只需更新Schema和向量库
2. **可控**：规则引擎确保业务约束
3. **可解释**：每一步操作都有明确指令
4. **可迭代**：对话历史可用于模型优化
