# 智能营销交互流程 - 技术实现设计

**最后更新时间**：2026-03-25 15:40 (北京时间)

## 一、整体架构概览

系统整体架构分为六层：

1. **用户输入层**：接收用户自然语言输入的活动需求、目标和约束条件
2. **大模型工作流编排层**：包含意图识别、任务分解、数据路由、执行编排四个子模块
3. **数据查询层**：生成结构化查询（SQL/DSL）、执行数据分析引擎、调用外部 API 获取标签和行为数据
4. **分析结果聚合层**：大模型对数据进行语义化解读与洞察提炼
5. **营销策略生成层**：基于分析结果生成可执行的营销策略
6. **策略输出与执行层**：对接 MA 系统、支持人工审核和 A/B 测试配置

数据流向：用户输入 → 意图识别 → 五层分析 → 洞察聚合 → 策略生成 → 执行配置 → MA 系统部署

## 二、用户输入问题 - 解析与结构化

### 2.1 输入解析工作流

输入解析分为三个步骤：

**Step1 预处理**
- 模型类型：小模型/规则
- 具体方法：正则表达式、字符串处理
- 文本清洗：去除特殊字符、标准化标点、统一时间格式
- 实体识别准备：大小写规范化、全角半角转换

**Step2 关键信息抽取（大模型 NER）**
- 模型类型：大模型
- 具体方法：NER 实体识别、语义理解
- 活动主题：字符串匹配 + 语义边界识别
- 活动周期：日期实体识别后标准化为 ISO8601 格式
- 商品范围：品牌名实体 + 价格条件解析
- 业务目标：目标类型分类 + 量化指标提取
- 目标客群：客群描述映射到标准化标签

**Step3 意图校验与补充**
- 模型类型：大模型
- 具体方法：逻辑推理、缺失检测
- 必填字段校验：主题、周期、目标、客群
- 缺失信息追问生成：当关键字段缺失时自动生成澄清问题
- 歧义检测：当检测到多个可能解释时生成确认请求

### 2.2 大模型 Prompt 配置要点

输入解析器的 Prompt 需定义以下内容：
- 提取字段定义：activity_theme、activity_period、product_scope、business_objectives、target_audience
- 输出格式要求：标准 JSON 格式、无法确定的字段设为 null、日期统一转换、品牌名标准化映射
- 用户输入占位符用于动态注入

### 2.3 输出数据结构

解析后的输出包含：
- 活动元数据：活动 ID、主题、周期（含起止日期和持续天数）
- 商品范围：品类列表、品牌列表、价格条件（含最低价、货币、比较条件）
- 业务目标：目标类型、描述、指标名称、适用范围
- 目标客群：客群类型、定义特征列表
- 校验状态和置信度评分

## 三、客群核心维度深度分析 - 详细实现

### 3.1 第一层：客群基础属性与核心特征画像分析

**分析目标**：
1. 这个人群是谁：明确客群的年龄、地域、职业、消费能力等基础属性
2. 有什么显著不同：找出客群相比全量用户在会员等级、品牌偏好、产品持有等方面的差异化特征

#### 3.1.1 数据查询配置

**数据源**
- 用户基础标签表（dim_user_profile）
- 会员等级表（dim_member_level）
- 订单事实表（fact_order_detail）
- 产品信息表（dim_product_info）

**查询逻辑 - Step1 客群圈选**
- 模型类型：小模型/规则
- 具体方法：SQL 规则引擎
- 解析活动目标客群描述：根据输入的自然语言需求，映射到对应的用户筛选条件
- 解析产品相关条件：筛选持有指定品类、品牌、价格区间产品的用户，或处于特定使用周期的用户
- 解析消费能力条件：筛选会员等级或历史消费达到指定阈值的用户
- 输出：临时客群表

**查询逻辑 - Step2 人群画像分析**
- 模型类型：小模型
- 具体方法：统计聚合（GROUP BY）
- 人口属性：年龄段分布、性别分布、地域分布（按城市等级）、职业分布
- 消费能力：会员等级分布、累计消费分段、平均客单价、分期偏好、促销敏感度标签
- 产品持有特征：持有品类分布、持有品牌分布、产品使用时长、产品状态（如损耗程度）、关联产品持有数

**查询逻辑 - Step3 显著性差异分析**
- 模型类型：小模型
- 具体方法：统计检验（TGI、Uplift）
- 对比基准：全量用户或同价位段用户
- 统计方法：TGI、uplift等
- 输出：TOP N 显著差异特征

#### 3.1.2 大模型洞察生成

- 模型类型：大模型
- 具体方法：数据解读、洞察提炼
- 分析维度：人口属性特征总结、消费能力分层、品类/品牌偏好分析、产品生命周期（如复购时机、换新时机等）
- 输出要求：每个维度 3-5 条核心洞察、洞察需有数据支撑、识别显著特征、输出结构化 JSON

#### 3.1.3 输出结构

- 客群规模
- 人口属性洞察列表（含洞察描述、支撑数据、重要性评分）
- 消费能力洞察列表
- 品类/品牌偏好洞察列表
- 产品生命周期信号列表（如复购时机、换新时机等）
- 显著差异特征列表

### 3.2 第二层：客群全链路行为与转化潜力分析

**分析目标**：
1. 用户离下单有多远：识别客群当前所处的决策阶段，是刚有购买念头还是已经准备下单
2. 有什么显著不同：找出客群特有的流失节点和转化瓶颈，比如哪些环节比全量用户更容易放弃

#### 3.2.1 数据查询配置

**数据源**
- 用户行为埋点表（fact_user_behavior_log）
- 搜索日志表（fact_search_log）
- 订单全量表（fact_order_detail）
- 客服会话表（fact_cs_session）
- 售后数据表（fact_after_sales）

**查询逻辑 - Step1 产品浏览与意向行为**
- 模型类型：小模型
- 具体方法：统计聚合
- 时间窗口：近 30 天、近 90 天
- 浏览产品分布：按品类、品牌、价格区间统计
- 浏览深度指标：浏览频次、停留时长、访问深度
- 意向行为转化：加购、收藏、预约、分享行为计数
- 输出：用户意向强度评分

**查询逻辑 - Step2 转化流失节点定位**
- 模型类型：小模型 + 大模型
- 具体方法：统计聚合 + 文本聚类
- 流失场景识别：详情页跳出、加购未下单、咨询后未成交、价格页跳出
- 流失原因聚类：从客服会话文本提取高频问题（大模型文本聚类）、从搜索日志提取未满足需求
- 输出：流失节点分布 + 高频流失原因 TOP10

**查询逻辑 - Step3 历史消费转化行为**
- 模型类型：小模型
- 具体方法：统计聚合
- 历史购买明细：历次购买时间、品牌、产品、价格
- 复购/换新周期规律：计算相邻购买时间间隔的平均值
- 连带消费偏好：配件购买率、延保购买率、分期渗透率
- 输出：复购/换新周期预测模型输入特征

**查询逻辑 - Step4 场景化刚需行为**
- 模型类型：小模型
- 具体方法：关键词匹配、规则引擎
- 场景标签识别：职场晋升、婚庆场景、企业采购、年终奖发放
- 识别逻辑：基于搜索/浏览关键词、账户类型、时间节点和行为模式
- 输出：场景标签及置信度

#### 3.2.2 意向强度评分模型

- 模型类型：小模型
- 具体方法：加权评分模型、时间衰减函数

评分计算逻辑：
- 权重配置：30 天浏览频次（10%）、平均停留时长（15%）、加购行为（25%）、收藏行为（15%）、预约行为（25%）、试用/估价行为（20%）、比价行为（15%）
- 时间衰减因子：越近的行为权重越高，使用指数衰减函数
- 评分计算：各行为值归一化后乘以衰减权重，求和后放大到 0-100 分

意向等级划分：
- 强意向（70-100 分）：多次深度浏览 + 加购/预约
- 中意向（40-70 分）：频繁浏览 + 比价/测评查看
- 弱意向（0-40 分）：偶尔浏览

#### 3.2.3 流失原因文本分析

- 模型类型：大模型
- 具体方法：文本聚类、主题提取
- 分析任务：主题聚类识别流失原因类别、提取具体顾虑点、统计频次和占比、评估影响程度
- 流失原因分类体系：价格相关、产品相关、服务相关、竞品对比、其他
- 输出结构：流失原因聚类列表（含类别、频次、占比、影响等级、具体顾虑点）、可执行洞察列表

#### 3.2.4 输出结构

第二层分析输出包含：
- 意向分布：各等级用户数和占比
- 浏览洞察列表
- 流失分析：TOP 流失节点、流失原因聚类
- 复购/换新周期预测：平均周期月数、待复购用户数、复购窗口分布
- 场景标签列表

### 3.3 第三层：客群触点偏好与交互响应分析

**分析目标**：
1. 什么时候、用什么渠道触达：找到客群最愿意打开消息的渠道和时间段
2. 有什么显著不同：找出客群相比全量用户对哪些渠道更敏感、对什么内容形式更感兴趣

#### 3.3.1 数据查询配置

**数据源**
- 触达历史表（fact_touchpoint_log）
- 触点交互表（fact_interaction_detail）
- 用户活跃日志表（fact_user_active_log）
- A/B 测试结果表（fact_ab_test_result）

**查询逻辑 - Step1 触点覆盖与偏好**
- 模型类型：小模型
- 具体方法：统计聚合、排序
- 触达渠道列表：APP PUSH、企业微信消息、短信、邮件、小程序订阅消息、电话外呼
- 效果指标计算：触达率、打开率、点击率、转化率、渠道 ROI
- 输出：各渠道效果排名 + 用户偏好渠道识别

**查询逻辑 - Step2 触达时机与频次耐受度**
- 模型类型：小模型
- 具体方法：统计聚合、趋势分析、拐点检测
- 活跃时段分析：按小时和星期统计活跃用户数，识别高峰时段 TOP5
- 响应率时段分析：统计各时段触达的响应率，找出响应率最高的时段窗口
- 频次耐受分析：统计历史触达频次分布，分析响应率随频次变化的趋势，识别拐点
- 输出：最佳触达时段 + 频次耐受阈值

**查询逻辑 - Step3 多触点转化漏斗**
- 模型类型：小模型
- 具体方法：漏斗转化率计算、PrefixSpan 序列模式挖掘
- 漏斗定义：触达→打开→点击→访问详情页→加购→下单
- 分析维度：各层级转化率和流失率、不同渠道组合的转化差异
- 最优路径挖掘：使用序列模式挖掘算法识别高转化用户的触点序列模式
- 输出：转化漏斗 + 最优触点序列

**查询逻辑 - Step4 内容形式偏好**
- 模型类型：小模型
- 具体方法：统计聚合、排序
- 内容类型：图文详情、短视频评测、直播带货、一对一话术、对比表格
- 效果指标：完读率/完播率、互动率、转化率
- 输出：内容形式效果排名

#### 3.3.2 输出结构

第三层分析输出包含：
- 渠道效果排名列表（含触达率、打开率、点击率、转化率、ROI、用户偏好评分）
- 最佳触达时机：高峰时段、高峰日期、最佳响应窗口
- 频次耐受：最佳周频次、耐受阈值、过频流失率、建议
- 转化漏斗：各阶段用户数和转化率、瓶颈节点、优化建议
- 内容偏好排名

## 四、营销策略生成引擎

### 4.1 策略生成工作流

**输入**
- 活动元数据（parsed_input）
- 五层分析结果聚合（layer1 至 layer5）

**处理流程 - Step1 客群底层逻辑锚定**
- 模型类型：大模型
- 任务：综合五层分析，提炼客群核心定义与运营原则
- 具体方法：数据解读、洞察聚合、逻辑推理
- 输出：客群底层逻辑描述

**处理流程 - Step2 策略框架生成**
- 模型类型：大模型
- 任务：基于分析洞察生成策略框架
- 具体方法：策略库匹配、创意生成、逻辑推理
- 策略库匹配：从策略库中匹配适用策略模板、基于客群特征调整策略参数
- 输出：4 个差异化策略草案

**处理流程 - Step3 策略细节填充**
- 模型类型：大模型 + 小模型/规则
- 具体方法：模板填充、规则校验
- 配置生成：人群圈选条件（SQL 生成）、触达渠道及时机配置、内容模板匹配、权益配置、频控规则
- 输出：策略执行配置 JSON

**处理流程 - Step4 策略验证与优化建议**
- 模型类型：小模型/规则 + 大模型
- 具体方法：规则校验、逻辑推理
- 验证项：人群覆盖量级是否合理、触达频次是否合规、权益预算是否可控、技术实现是否可行
- 输出：验证报告 + 优化建议

### 4.2 策略生成 Prompt

策略生成器 Prompt 需定义：
- 输入信息：
  - 活动元数据（主题、周期、商品范围、业务目标）

  - 合规约束（第五层输出的排除规则和频次限制）
- 策略生成要求：差异化定位、基于分析洞察、可执行、策略间协同
- 策略框架参考：分层运营策略、行为触发策略、忠诚度策略、流失拦截策略
- 每个策略输出结构：策略名称、核心思路、目标用户分层、核心动作、预期价值
- 输出格式：客群逻辑锚定、策略列表

策略执行配置包含以下字段：
