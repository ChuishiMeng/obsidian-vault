# KDD Cup 2026 Data Agents 竞赛

## 赛事概述

KDD Cup 2026 聚焦于 **Data Agent（数据智能体）** 领域，考察 AI Agent 在复杂数据分析场景下的核心能力。

---

## 核心能力考察

| 能力 | 描述 |
|------|------|
| **智能分解与规划** | 将高层级分析问题，自主拆解为多步骤、可执行的精细化计划 |
| **灵活工具调用** | 在每个推理环节，精准选择并调用 Python 脚本、SQL 查询、API 接口等各类工具 |
| **异构数据融合推理** | 在结构化表格、非结构化文档、图表乃至多模态数据源之间，进行无缝、高效的交叉推理 |
| **结果综合与决策** | 将多步骤的中间结果进行智能综合，最终给出准确、可信的答案 |

---

## 比赛阶段

### Phase 1（基础模态）

重点考察 Agent 对以下数据类型的理解与综合推理能力：

- 结构化数据库（SQL）
- 半结构化文件（JSON/CSV）
- 非结构化文档（TXT/Markdown）

这是目前主流 Agent 的核心能力边界，旨在筛选出具备稳健基础能力的优秀团队。

### Phase 2（进阶挑战）

引入更具挑战性的异构模态：

- 数据图表
- 音频
- 复杂长文本报告

进一步探索 Data Agent 在处理碎片化信息时的上限。

## 示例任务

### 数据源

| 文件 | 描述 |
|------|------|
| `sales_transactions.sqlite` | 原始交易数据库，包含数百万条流水记录，但缺乏具体的业务逻辑映射 |
| `region_mapping.json` | 半结构化配置文件，记录了最新的地区代码（region_id）与具体业务区域名称的对应关系 |
| `Company_Ops_Manual.md` | 非结构化标准运营手册，定义了本季度"预算目标"的计算公式，以及各品类的标准分类 |
| `Market_Report_Q3.md` | 市场分析报告，包含了对各地区宏观政策变动的定性描述，是分析业绩增长驱动因素的关键 |

### 任务流程

1. **知识对齐与映射**：从 Company_Ops_Manual.md 中提取核算公式，并结合 region_mapping.json 将数据库中的 ID 映射为真实的业务区域
2. **多源数据整合**：基于提取的公式与映射逻辑，对 sales_transactions.sqlite 执行多表关联查询与复杂计算，筛选出符合条件的地区名单
3. **深度归因分析**：结合 Market_Report_Q3.md 中的定性描述，对筛选出的地区进行业绩增长原因的自动化归因，最终输出经过跨模态推理后的分析结果

## 赛道设置

### Phase 1

- 单一主赛道
- 所有注册队伍将在统一的公共排行榜上竞技
- 通过自动化评估进行排名

### Phase 2

成功晋级的队伍将进入第二阶段，并可自由选择：

| 赛道 | 目标 |
|------|------|
| **Leaderboard Subtrack（排行榜赛道）** | 追求极致准确率，面对包含数据图像、数据视频等新模态的基准测试 |
| **Creative Subtrack（创意赛道）** | 注重系统设计与用户体验，构建成熟、交互友好且决策透明的 Data Agent 系统 |
