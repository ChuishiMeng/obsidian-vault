## 一、项目背景与目标

### 1.1 为什么需要二次开发？

**原始 MoltBot 的局限**：
- 主要面向海外用户
- 接入 WhatsApp 等国外应用
- 模型以 Claude、OpenAI 为主

**国内适配需求**：
- 接入国内通信软件（钉钉、飞书）
- 支持国产大模型（DeepSeek）
- 中文交互体验优化

### 1.2 二次开发目标

| 目标 | 说明 |
|------|------|
| **功能汉化** | 全面中文化界面 |
| **工具替换** | 国外工具 → 国内工具 |
| **模型适配** | 支持国产开源模型 |
| **体验优化** | 聊天界面本地化 |

## 三、技术实现要点

### 3.1 沙盒环境设计

**核心理念**：安全隔离执行

```
用户请求 → 沙盒环境创建 → 代码执行 → 结果返回 → 环境销毁
```

**优势**：
- 保护主系统安全
- 支持任意代码运行
- 失败自动清理

### 3.2 Computer Use + Browser Use

| 能力 | 说明 |
|------|------|
| **Computer Use** | 操作本地工具、文件、命令行 |
| **Browser Use** | 控制浏览器、访问网页、模拟人工 |

**实际应用**：
- 自动化数据采集
- 网站测试
- 信息检索

### 3.3 长期运行一致性

**问题**：复杂任务需要多步调用，中途可能失败

**解决方案**：
```
执行 → 遇到问题 → 自动诊断 → 尝试修复 → 继续执行
```

**典型场景**：
- 依赖不兼容 → 自动调整版本
- 文件无法读取 → 检查权限
- 反爬措施 → 更换策略
