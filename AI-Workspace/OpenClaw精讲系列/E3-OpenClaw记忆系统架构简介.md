### 三、Skills技术详解

#### 什么是Skills？
Skills的本质是让大模型"知道"如何完成某项任务。核心逻辑：
> **"能不能做，取决于知不知道"**

#### Skills vs Function Calling 对比

| 对比项 | Function Calling | Skills |
|--------|------------------|--------|
| 调用时机 | 用户提到相关需求时 | 用户提到相关需求时 |
| 内容形式 | 外部函数定义 | SKILL.md文档 + 可选脚本 |
| 触发机制 | 检查系统消息中的函数定义 | 检查系统消息中的Skills元数据列表 |
| 加载方式 | 直接可用 | 通过ReadFile工具按需加载完整文档 |

#### Skills工作流程

```
用户请求 → Agent检查系统消息 → 发现匹配的Skill →
调用ReadFile读取SKILL.md → 理解操作步骤 →
使用内置工具执行任务 → 返回结果
```

#### 关键技术点：ReadFile作为内部工具

Skills实现的核心在于：
- 大模型原生无法读取文件
- 需要一个**ReadFile内部工具**来加载Skills文档
- 这个工具可以用Python沙盒环境简单实现

### 四、Skills的三种实现层级

#### Level 1: 纯指令型
仅包含操作说明，依赖Agent内置工具完成
```markdown
---
name: weather-query
description: 查询天气信息
---
使用命令行工具执行：
curl wttr.in/{城市名}
```

#### Level 2: 指令+脚本型
包含专用脚本，用于复杂任务
```
skills/
└── pdf-to-markdown/
    ├── SKILL.md
    └── convert.py    # 专用转换脚本
```

#### Level 3: 智能组装型
Skills可自主调用其他Skills

### 五、自我优化与迭代机制

#### 为什么OpenClaw能"越用越聪明"？

```
┌──────────────────────────────────────────────────┐
│            自我进化核心原理                        │
├──────────────────────────────────────────────────┤
│  1. 记忆文档存储在本地                            │
│  2. Agent有权限访问自己的Skills和记忆文件         │
│  3. 用户反馈错误 → Agent修改SKILL.md             │
│  4. 下次执行时读取更新后的文档 → 不再犯错         │
└──────────────────────────────────────────────────┘
```

#### 实现自进化的三要素

1. **内置工具**：提供修改文件的能力
2. **本地存储**：记忆文档完全本地化
3. **Skills架构**：支持动态更新

## 总结

OpenClaw记忆系统架构的三大支柱：

1. **工具生态**：四大内置工具覆盖文件、命令、网络、浏览器
2. **Skills热插拔**：通过Function Calling按需加载，支持自主创建修改
3. **记忆系统**：本地存储 + Skills文档 = 自我进化能力

这套架构是2026年Agent开发的核心技术方向，适用于通用智能体和垂直领域智能体的开发。
