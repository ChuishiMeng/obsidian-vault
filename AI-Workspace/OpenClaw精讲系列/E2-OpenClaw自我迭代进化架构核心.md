### 二、内置工具集详解

OpenClaw内置四大核心工具：

| 工具 | 功能 | 用途 |
|------|------|------|
| **文件操作工具** | 读写本地文件和文件夹 | 清理桌面、读取文件、整理文档 |
| **命令行执行工具** | 执行Shell命令 | 运行脚本、安装库、执行爬虫 |
| **网络搜索工具** | Fetch/Brave Search | 获取网页信息、网络搜索 |
| **浏览器自动化工具** | 操作浏览器 | 网页爬取、模拟人工操作 |

### 三、Skills系统技术原理

#### 3.1 Skills的本质

Skills是Agent执行某项工作的操作指南，核心是Markdown文档。

**Skills文档结构**:
```
skills/
└── weather-query/
    ├── SKILL.md       # 主要操作指南
    └── scripts/       # 可选：执行脚本
        └── query.py
```

#### 3.2 Skills的调用流程

```
用户: "查天气"
    ↓
Agent检查系统消息中的Skills列表
    ↓
发现"weather-query" Skills可用
    ↓
Function Calling: 调用ReadFile工具
    ↓
读取 SKILL.md 文件内容
    ↓
根据文档说明执行操作（如使用curl发请求）
    ↓
返回天气信息
```

#### 3.3 Skills与Function Calling的关系

**关键理解**: Skills调用本质上是一次Function Calling

1. Skills的元数据（名称、描述、路径）拼接到系统消息
2. 当用户提到相关需求时，模型发现Skills可用
3. 模型通过Function Calling调用ReadFile工具读取Skills文档
4. 读取到的文档内容作为工具返回结果（Tool Response）
5. 模型根据文档内容执行后续操作

### 四、自我进化机制

#### 4.1 核心原理

OpenClaw的"越用越聪明"并非模型自身变聪明，而是：

1. **本地存储**: 所有记忆文件、Skills文档都存储在本地
2. **可修改权限**: Agent有能力读取和修改自己的文档
3. **持久化更新**: 修改后永久生效

#### 4.2 自我优化的实现

**场景**: 用户发现查天气总是出错，建议换一个API

```
用户: "查天气老是失败，咱们换个API吧，用B网站代替A网站"

Agent执行流程:
1. 查找天气Skills的位置
2. 使用文件编辑工具修改SKILL.md
3. 将API地址从A改为B
4. 更新Memory.md记录这次修改

下次查询天气:
- 读取更新后的SKILL.md
- 使用B网站的API
- 成功获取天气
```

#### 4.3 三大条件

实现自我进化需要满足：

| 条件 | 说明 |
|------|------|
| **内置工具** | 拥有文件读写、命令执行等能力 |
| **本地存储** | 记忆和Skills存放在本地，可被访问 |
| **Skills架构** | 支持标准的Agent Skills体系 |

### 五、对开发者的启示

1. **通用智能体架构**: 这套架构可复制到任何领域
2. **自我优化能力**: 适用于所有垂直领域的Agent开发
3. **2026年热门方向**: Skills + 本地记忆 + 内置工具

## 总结

OpenClaw的自我进化能力来源于：
- **可修改的本地记忆系统**
- **标准化的Skills热插拔架构**
- **强大的内置工具集**

这三者的组合让Agent能够从错误中学习，永久性地改进自己的行为，实现"越用越聪明"的效果。
