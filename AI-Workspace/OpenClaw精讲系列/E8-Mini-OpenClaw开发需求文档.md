## 详细内容总结

### 一、需求文档的作用

当使用AI编程工具（如Cursor）开发项目时：

```
需求文档 → AI理解项目 → 自动生成开发规范 → 逐步完成开发
```

**关键认知**：
- 使用AI工具开发 ≠ 不需要理解项目
- 必须对功能、技术架构有深刻理解才能写好需求文档
- 需求文档是引导AI完成开发的"指挥棒"

### 二、项目定位

```
Mini OpenClaw - 轻量级AI Agent系统

定位: 基于Python重构的轻量且高度透明的AI Agent系统
目标: 复刻并优化OpenClaw（原名MoltBot/Multiple CloudBot）的核心体验

核心特性（本版本重点）:
├── 文件级记忆管理
├── 技能热插拔（Skills）
└── 透明可控（全流程可追踪）
```

### 三、需求文档结构

#### 第1部分：基本架构与项目介绍

| 项目 | 说明 |
|------|------|
| 后端语言 | Python |
| Web框架 | FastAPI |
| Agent引擎 | LangChain 1.2+ (create_react_agent) |
| LLM接口 | 统一使用 create_react_agent API |
| RAG引擎 | LlamaIndex (混合检索) |

**重要说明**：
```markdown
⚠️ 严禁使用旧版 LangChain Agent API
原因: 很多大模型会默认使用早期 Agent 执行方式
      这是一个重大的 API 兼容性问题
```

**混合检索代码示例**: 建议直接提供代码片段，让AI更好理解实现方式。

#### 第2部分：内置工具（Tools）

需要明确列出所有内置工具：

| 工具名称 | 功能说明 |
|----------|----------|
| Terminal Tool | 命令行操作工具 |
| Python REPL | Python代码解释器 |
| Fetch Tool | 网络信息获取 |
| File Tool | 文件读取与操作 |
| RAG Search | 知识库检索工具 |

#### 第3部分：Agent Skills系统

**功能顺序**: 先基础功能，后复杂功能

Skills系统说明要点：
```
1. 本质定义: 一个文件夹 = 一个技能
2. 文档结构: SKILL.md + 元数据
3. 元数据格式: name + description
4. Snapshot生成: 自动拼接成System Message
5. 调用流程: 感知列表 → Function Call决策 → 读取完整文档 → 执行操作
```

**调用流程图**:
```
System Prompt 感知 Skills 列表
        ↓
   Function Call 决策
        ↓
   选择具体 Skill
        ↓
   读取 SKILL.md 完整内容
        ↓
   根据文档指导执行后续操作
```

#### 第4部分：记忆管理系统设计

```
记忆管理核心原则:
├── 所有记忆存储为本地文件
├── Prompt由6部分构成
├── 会话存储为JSON格式
└── 支持Summary压缩与RAG检索
```

**Prompt构成**:
1. SOUL.md - 人格定义
2. IDENTITY.md - 身份设定
3. AGENTS.md - Agent说明书
4. USER.md - 用户信息
5. MEMORY.md - 长期记忆
6. skills-snapshot - 技能列表

#### 第5部分：后端API接口规范

如果熟悉FastAPI可自行编写，否则可让大模型生成：
- 大模型对后端接口设计非常清楚
- 只需明确业务需求，接口规范可自动生成

#### 第6部分：前端开发

**现代开发方式**:
- UIUX Pro Max
- Bolt.new
- Cursor + Figma

**本项目采用**: 传统方式 - 明确前端技术栈和风格规范

**重要**: 项目目录结构需提前确定，否则开发过程不可控

### 四、Cursor使用技巧

#### 1. 初始化项目

```bash
# 步骤1: 将需求文档放入 .cursor/ 文件夹
mv requirements.md .cursor/

# 步骤2: 输入初始化命令
# 在 Cursor 中: Ctrl+L → 输入 "@初始化项目" 或类似指令
```

#### 2. CLAUDE.md 自动生成

Cursor初始化过程：
```
1. 读取项目所有文件
2. 发现只有需求文档
3. 理解需要从0到1开发
4. 自动提取需求文档核心内容
5. 生成 CLAUDE.md（最高开发准则）
```

**CLAUDE.md的作用**:
- AI本次运行的System Prompt
- 自动生成的开发规范
- 确保开发方向一致
