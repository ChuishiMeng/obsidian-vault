## 六重境界详解

### 第一重：提示工程师 (Prompt Engineer)

**特征：**
- 把 Claude Code 当作简单工具使用
- 直接下达指令，如"给我搭建一个网站"
- 单线互动关系

**问题：**
- 产出平庸（"千篇一律的垃圾"）
- 回归平均值（Regression to the Mean）
- 大多数 AI 生成的网站都用紫色界面和类似图标

**核心技能：**
1. 编写清晰具体的提示词
2. 明确最终目标和子功能
3. 学会阅读和评估 Claude 的输出
4. 了解基本的终端命令和软件工程术语

**突破关键：** 从"命令"转向"邀请协作"

---

### 第二重：规划者模式 (Planner Mode)

**特征：**
- 启用计划模式（Plan Mode）
- 双向互动，不再是单方面指令
- Claude 通过提问来协作

**核心技能：**
1. 熟练掌握计划模式
2. 学会提出引导性问题
   - "我是不是漏掉了什么？"
   - "可能会有什么意想不到的后果？"
   - "如果是行业设计专家，他们会怎么考虑？"
3. 让 Claude 扮演对抗性角色（Devil's Advocate）

**突破关键：** 从"文字指令"转向"引入背景信息"

---

### 第三重：上下文工程师 (Context Engineer)

**特征：**
- 开始引入外部信息（文件、截图、具体示例）
- 管理上下文窗口

**核心概念：上下文腐化 (Context Corruption)**

- 上下文窗口填充越多，AI 表现越差
- 腐化触发点：约 10-12 万 tokens（50-60%）
- 即使窗口扩大到 100 万 tokens，腐化点仍在 10 万左右

**核心技能：**
1. 监控上下文窗口状态（/+context 命令）
2. 及时重置窗口（/+reset）
3. 创建常驻状态栏显示窗口使用情况
4. 手动总结关键内容带到新窗口

**突破关键：** 从"管理上下文"转向"选择正确工具"

---

### 第四重：工具整合者 (Tool Integrator)

**特征：**
- 接触 MCP 服务器、插件、框架
- 像"进了糖果店的孩子"

**警告：**
- 工具过多反而降低效率
- 需要判断哪些工具真正有用

**核心技能：**
1. 评估和选择合适的 MCP 服务器
2. 避免工具过度配置
3. 理解工具之间的协同效应

**突破关键：** 从"使用工具"转向"设计系统"

---

### 第五重：架构师 (Architect)

**特征：**
- 设计系统架构
- 定义工作流程和规范

**核心概念：**
- 建立可复用的工作模式
- 设计约束让 Claude 更高效

**突破关键：** 从"单次任务"转向"持续优化"

---

### 第六重：驾驭者 (Harness Engineer)

**特征：**
- 完全掌握 Claude Code 的协作模式
- 能够同时管理多个 Agent/任务
- 实现 10 倍效率提升

**核心技能：**
1. 深度优先的工作方式
2. 设计清晰可执行的约束
3. 建立反馈循环和审核机制
4. 持续垃圾收集（重构和优化）

---

## 关键洞察

### 上下文腐化

> "上下文窗口填充的内容越多，AI 模型的表现越糟糕"

- 腐化触发点：约 10-12 万 tokens
- 与窗口总大小无关（100 万窗口也会在 10 万腐化）
- 解决方案：监控、重置、手动总结

### 工具选择

> "少即是多"

- AGENTS.md 等上下文文件可能拉低成功率
- 提供"一些"上下文，但不要"过多"
- 判断 Claude 真正需要什么

### 协作模式转变

| 阶段 | 关系 | 互动方式 |
|------|------|----------|
| 1 | 工具 | 命令 |
| 2 | 助手 | 提问 |
| 3 | 伙伴 | 提供背景 |
| 4 | 团队 | 工具协同 |
| 5 | 架构师 | 系统设计 |
| 6 | 驾驭者 | 自主管理 |

## 实践建议

### 立即行动

1. **启用计划模式**：开始双向协作
2. **监控上下文**：创建状态栏显示窗口使用
3. **学会提问**："我是不是漏掉了什么？"
4. **精简工具**：只保留真正有用的 MCP

### 进阶路径

1. 建立 AGENTS.md（但要精简）
2. 设计可复用的工作流
3. 培养"对抗性"思维（让 Claude 挑战你的方案）
4. 实现多 Agent 协作

## 与 Harness Engineering 的关联

这个视频的"六重境界"与 OpenAI 的 Harness Engineering 理念高度一致：

| 视频境界 | Harness 概念 |

- 人类掌舵，AI 执行
