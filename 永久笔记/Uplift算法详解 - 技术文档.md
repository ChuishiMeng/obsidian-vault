# Uplift 算法详解

> 整理自 2026-03-19 技术讨论
> 适用场景：AB 实验分析、营销人群筛选、个性化策略

---

## 一、什么是 Uplift 算法

### 核心概念

Uplift 算法（增量模型）用于预测**干预对个体的因果效应**——简单说：

> "给这个用户发优惠券，他会不会因为券而多买？"

### 数学定义

$$\tau(x) = E[Y | X=x, T=1] - E[Y | X=x, T=0]$$

- **Uplift** = 干预后转化率 - 不干预转化率
- **目标**：找出"干预才买"的用户（Persuadables）

### 用户分群（四象限）

```
                    会转化
                      ↑
   Lost Causes     Persuadables  ← 发券才买 ✅
   (怎么都不买)     (发券有用!)
                      |
   Sure Things      Sleeping Dogs
   (本来就买)        (发券反而不买)
                      ↓
                    不会转化
```

| 人群类型 | 特征 | 策略 |
|---------|------|------|
| **Persuadables** | 发券才买 | 重点发券 ✅ |
| **Sure Things** | 本来就买 | 不需要发券 |
| **Lost Causes** | 怎么都不买 | 不浪费资源 |
| **Sleeping Dogs** | 发券反而不买 | 别打扰 ⚠️ |

## 二、Uplift 在 AB 实验中的应用

### 传统 AB 分析 vs Uplift 分析

| 维度 | 传统方式 | Uplift 方式 |
|------|---------|------------|
| 输出 | 平均提升（ATE） | 个体提升（ITE/CATE） |
| 粒度 | "整体提升 5%" | "这 20% 用户提升 25%" |
| 策略 | 一刀切 | 精准分层 |

### 能回答的问题

| 问题 | 传统 AB | Uplift |
|------|--------|--------|
| 整体有效吗？ | ✅ | ✅ |
| 对谁有效？ | ❌ | ✅ |
| 对谁有害？ | ❌ | ✅ |
| 下次该对谁发？ | ❌ | ✅ |

### 应用场景

1. **实验后人群细分**：预测每个用户的 uplift 分数，分层分析
2. **发现负面效应**：揪出 Sleeping Dogs，避免打扰
3. **个性化策略部署**：只对 uplift > 阈值的用户上线策略

## 三、输入与输出

### 输入

```
用户特征 X = [
    历史购买次数,
    最近活跃天数,
    客单价,
    会员等级,
    品类偏好,
    ...
]
```

### 输出

```
uplift_score = P(转化|干预) - P(转化|不干预)
```

**一个数值**，表示"干预带来的增量概率"

### 训练数据需求

```
X, y, treatment
│   │     │
│   │     └─ 是否在实验组（0/1）
│   └─ 是否转化（0/1）
└─ 用户特征
```

## 四、特征重要性分析

### 两种特征重要性

| | 传统模型 | Uplift 模型 |
|--|---------|------------|
| 问题 | 谁会转化？ | 谁因为干预才转化？ |
| 特征重要性 | 预测转化的特征 | 预测**增量**的特征 |

**Uplift 的特征重要性更精准**——告诉你"什么特征让干预更有效"

### 分析方法

1. **树模型的 feature_importances_**：直接获取
2. **SHAP 分析（推荐）**：看重要性 + 方向
3. **Permutation Importance**：打乱特征看影响
4. **分层对比分析**：按 uplift 分组看特征差异

### 实际解读示例

```
特征重要性：
1. 最近活跃天数    0.32  ← 最近活跃的用户，发券效果更好
2. 历史优惠券使用率 0.25  ← 爱用券的人，发券有效
3. 会员等级        0.18  ← 中等级会员最敏感
4. 客单价          0.08  ← 低影响
```

## 五、训练与推理算法

### 方法对比

| 方法 | 核心思想 | 适用场景 | 通俗比喻 |
|------|---------|---------|---------|
| **T-Learner** | 两个模型分别预测 | 处理效应明显 | 两个老师分别教 |
| **S-Learner** | 干预作为特征 | 快速原型 | 一个老师全看 |
| **X-Learner** | 交叉学习反事实 | 处理效应弱 | 互相猜反事实 |
| **R-Learner** | 残差回归 | 理论保证 | 找"意外" |
| **Uplift Tree** | 直接优化 uplift 分裂 | 可解释性需求 | 专门找"效果差异" |

### T-Learner（双模型）

**思想**：分别训练两个模型，预测结果相减

```
训练过程：
├─ 实验组模型：μ₁(x) = 预测"被干预时的转化概率"
└─ 对照组模型：μ₀(x) = 预测"不被干预时的转化概率"

推理过程：
uplift = μ₁(x) - μ₀(x)
```

**优缺点**：
- ✅ 简单直观，可用任意模型
- ❌ 样本利用率低，两个模型误差叠加

### S-Learner（单模型）

**思想**：把干预作为特征，训练一个模型

```
训练过程：
X' = [X, T]  # 把 T 作为第 n+1 个特征

推理过程：
prob_if_treated = model.predict([x, 1])
prob_if_control = model.predict([x, 0])
uplift = prob_if_treated - prob_if_control
```

**优缺点**：
- ✅ 样本利用率高，只需训练一个模型
- ❌ 当干预效应很小时，模型可能忽略 T 特征

### X-Learner（交叉学习）

**思想**：利用对照组预测实验组的反事实，反之亦然

```
Step 1: 训练基础模型（类似 T-Learner）
Step 2: 计算个体处理效应（ITE）
Step 3: 训练效应模型
Step 4: 加权组合
```

**优缺点**：
- ✅ 样本利用充分，理论保证更强
- ❌ 实现复杂，需要估计倾向得分

### 方法选择建议

| 场景 | 推荐方法 | 理由 |
|------|---------|------|
| 快速原型 | S-Learner | 简单，样本利用充分 |
| 处理效应较强 | T-Learner | 直接建模差异 |
| 处理效应较弱 | X-Learner / R-Learner | 更精细的估计 |
| 需要可解释性 | Uplift Tree | 树结构直观 |
| 生产环境 | X-Learner（CausalML） | 稳定，成熟库 |

## 六、组合特征分析

### 方法对比

| 方法 | 输出 | 适用场景 |
|------|------|---------|
| **Uplift Tree** | 决策规则（if-then） | 最直观，直接汇报 |
| **SHAP 交互** | 交互强度数值 | 量化分析 |
| **分组统计** | 各组合的实际 uplift | 验证模型发现 |
| **规则提取** | 特征组合列表 | 部署到业务系统 |

### Uplift Tree 输出示例

```
根节点：全部用户（uplift = +3%）
    │
    ├─ 年龄 < 35？
    │   ├─ 是：uplift = +2%
    │   └─ 否：
    │       ├─ 活跃度 > 高？
    │       │   ├─ 是 + 历史用券 > 2次？
    │       │   │   ├─ 是：uplift = +15% ✅ 高价值人群
    │       │   │   └─ 否：uplift = +5%
    │       │   └─ 否：uplift = -2% ⚠️ 别发券
```

### Uplift 发现的组合效应

**传统分析**：
- 年龄 30-40：+5% lift
- 高活跃用户：+3% lift

**Uplift 能发现**：
- 年龄 30-40 **且** 高活跃 **且** 历史用过券：+15% lift
- 年龄 30-40 **且** 低活跃：+1% lift（无效）

## 七、寻找高敏感人群

### 筛选流程

```python
# 预测所有用户的 uplift 分数
uplift_scores = model.predict(X)

# 正向高敏感：uplift 高
positive_sensitive = users[uplift_scores > threshold_high]

# 负向高敏感：uplift 低（甚至为负）
negative_sensitive = users[uplift_scores < threshold_low]
```

### 阈值定义

```python
# 方法A：百分位切分
high_threshold = np.percentile(uplift_scores, 80)   # top 20%
low_threshold = np.percentile(uplift_scores, 20)    # bottom 20%

# 方法B：基于业务定义
high_threshold = 0.05   # uplift > 5% 认为值得干预
low_threshold = 0       # uplift < 0 认为会起反作用
```

### 人群分布示例

```
            uplift_mean  uplift_count  age_mean  activity_mean  
segment
负向敏感      -0.03         5000        45.2        5.1          
中性         +0.02        15000        38.5       12.3          
正向敏感      +0.08         5000        32.1       25.6          
```

### 业务应用建议

| 人群 | Uplift 范围 | 策略 |
|------|------------|------|
| **正向高敏感** | uplift > 80分位 | 重点发券，高 ROI |
| **负向高敏感** | uplift < 20分位 | 不打扰，省钱避坑 |
| **中性** | 中间区间 | 看预算 |

## 八、反向总结人群特征

### 方法一：平均值对比

**就像做体检报告对比**

```
高敏感人群体检报告：
├─ 平均年龄：32 岁
├─ 平均活跃度：25
└─ 平均用券次数：3 次/月

低敏感人群体检报告：
├─ 平均年龄：45 岁
├─ 平均活跃度：5
└─ 平均用券次数：0.8 次/月
```

### 方法二：效应量（差异大不大）

**判断差异是"很大"、"一般"还是"可以忽略"**

```
| Cohen's d | 效应大小 |
|-----------|---------|
| > 0.8     | 大效应  |
| > 0.5     | 中效应  |
| < 0.5     | 小效应  |
```

### 方法三：决策规则（自动找规律）

**就像玩"猜猜我是谁"游戏**

```
高敏感人群 = 活跃度高 + 爱用券 + 年龄小于 40
```

### 方法四：贡献度分析（谁最重要）

**就像团队绩效考核**

```
活跃度：★★★★★ 贡献最大
历史用券：★★★★ 贡献大
年龄：★★★ 有贡献
客单价：★ 贡献很小
```

### 完整流程

```
第一步：平均值对比 → 快速看出两组人的差异
第二步：效应量筛选 → 过滤掉"假差异"的特征
第三步：决策规则总结 → 写成一句话，给老板汇报
第四步：贡献度排序 → 告诉业务方，应该关注哪些指标
```

## 十、与你们现有方法的对比

### 你们现在的做法

```
AB 实验结果
    ↓
按特征分组：年龄、会员等级、活跃度...
    ↓
每组计算：实验组转化率 - 对照组转化率
    ↓
看哪个特征分组差异大
```

### 对比

| 维度 | 你们的方法 | Uplift 方法 |
|------|-----------|------------|
| 特征维度 | 一次看一个特征 | 所有特征同时建模 |
| 特征组合 | 需要手动交叉分析 | 自动发现交互效应 |
| 样本利用 | 分组后样本量变小 | 全量数据训练 |
| 边界划分 | 人工切分 | 自动学习最优边界 |
| 输出粒度 | 群体级别的 lift | 个体级别的 uplift |

### 建议组合使用

```
第一步：你们的方法
├─ 快速发现哪些特征可能有影响
└─ 确定大致方向

第二步：Uplift 建模
├─ 用所有候选特征训练
├─ 自动发现特征组合
└─ 输出个体 uplift 分数

第三步：SHAP 解释
├─ 验证你们的发现
└─ 发现新的洞察

第四步：决策
├─ 高 uplift 用户 → 激进策略
└─ 用你们的方法做汇报图表
```

## 总结

> **Uplift 的核心价值**：从"谁会转化"到"谁因为干预才转化"

**适用场景**：
- AB 实验后的人群细分
- 营销优惠券精准发放
- 推送人群筛选
- 避免给"本来就会买"的人浪费资源

**核心产出**：
1. 个体级别的 uplift 分数
2. 高/低敏感人群特征
3. 特征重要性排序
4. 决策规则（if-then）
